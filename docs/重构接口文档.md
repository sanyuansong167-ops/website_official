# 重构接口文档

本文档用于记录重构官网新加入的接口，每新增、修改、删除一个接口，必须同步更新本文档。

---

## 维护规则

* **接口变更必须更新**：新增接口、修改接口路径、修改请求参数、修改响应字段、修改错误码、删除接口时，必须同步更新本文档。
* **必须标注实现位置**：每个接口条目必须写明 Controller 文件路径和方法名，便于快速定位代码。
* **必须区分接口类型**：后台接口归入 Admin 接口，前台公开接口归入 Portal 接口。
* **必须记录鉴权要求**：每个接口必须说明是否需要登录、需要什么权限、是否可匿名访问。
* **必须记录数据可见性**：Portal 接口必须说明是否过滤隐藏和逻辑删除数据；草稿、审批、定时发布等状态仅在PRD明确引入后记录。
* **必须记录错误码**：接口可能返回的业务错误码必须写入对应条目，错误码按 `docs/错误码说明.md` 中的数字分段规则设置。
* **文档先于交付**：接口代码完成但本文档未更新，视为接口未完成。

## Portal 缓存自动化机制

所有 Portal 公开只读查询接口（`/portal/api/**` 下的 GET 接口）默认纳入 Redis 缓存自动化机制，由 `infrastructure.cache.PortalCacheSupport` 统一承接读取、写入与失效。

### 缓存 Key 命名规则

统一格式：`official:portal:{module}[:{segment}...]`，由 `PortalCacheKeyBuilder` 构造，业务代码禁止手写完整 Key。

### 读取链路（read-through）

1. 按 module/segment 构建缓存 Key
2. 从 Redis 读取；命中则反序列化返回，不再访问数据库
3. 反序列化失败（含 Redis 字节损坏抛 `SerializationException`、或转换为目标 VO 类型失败）视为缓存损坏：**主动删除坏缓存**并回源数据库，实现自愈
4. Redis 连接故障等非数据损坏异常时仅降级查库，不触发删除，不向前台报错
5. 未命中则查库、组装 VO、写回缓存

### 写入与 TTL

- 非空结果使用 `official.cache.default-ttl`（默认 10min，可按模块覆盖）
- 空结果使用 `official.cache.empty-result-ttl`（默认 5min），降低缓存穿透风险
- per-module 覆盖：`official.cache.portal-ttl-overrides.{module}` 可为指定模块单独设置 TTL；`{module}` 取缓存 Key 的**模块段**（`PortalCacheKeyBuilder.build` 的首参），与 Key 命名空间保持一致
  - 单段 Key：模块段即唯一段，如 `ai_cards`、`honors`、`site-config`
  - 多段 Key：模块段为首段，其下所有 segment 共享同一覆盖键，如 `home` 同时作用于 `official:portal:home:banner` 与 `official:portal:home:metrics`
- 所有缓存必须设置 TTL，不允许永久缓存

### 失效链路（after-commit + 延迟二次删除）

Admin 任意 Create/Update/Delete/显示隐藏/排序等写操作成功后：

1. 在数据库事务提交后（`afterCommit`）立即删除对应 Portal Key
2. 调度延迟二次删除（`official.cache.second-delete-delay`，默认 1s）兜底并发读旧值回填
3. 事务回滚时不得删除缓存
4. 缓存删除失败仅记日志，不回滚业务事务，由 TTL 兜底
5. 一次写操作影响多个 Key 时必须全部清除（如聚合接口依赖子模块）

### 失效责任归属

- 缓存失效由触发写操作的业务 Service 显式声明需失效的 Key
- 基础设施层只负责执行删除，不猜测 Key
- 聚合接口依赖关系由 application 层显式维护

### 不纳入缓存的接口

- Portal 表单提交、上传、下载、回调接口
- Admin 任意查询接口
- 登录、鉴权、权限类接口
- 限流计数、验证码等安全控制接口
- 线索后台看板等敏感数据查询

## Phase 2 接口

### 获取组件模板列表

* **接口类型**：Admin
* **请求方法**：GET
* **接口路径**：`/admin/api/page-builder/component-templates`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminComponentTemplateController.java#getTemplateList`
* **鉴权要求**：需角色：管理员
* **功能说明**：获取全部已启用的组件模板列表，按 `sort_order` + `id` 升序排列。
* **请求参数**：无
* **响应字段**：
    * `data`：`Array<ComponentTemplateVO>`，组件模板列表。
        * `id`：Long，模板 ID。
        * `componentCode`：String，组件唯一编码，如 `HeroBanner`。
        * `name`：String，组件名称。
        * `category`：String，组件分类（SHOW / STRUCTURE / BUSINESS / FORM）。
        * `schemaDefinitionJson`：Object，属性 Schema 定义 JSON。
        * `defaultPropsJson`：Object，默认属性 Props JSON。
        * `bindingCapabilityJson`：Object，数据绑定能力定义 JSON。
        * `status`：String，组件状态（ACTIVE / INACTIVE）。
        * `sortOrder`：Integer，排序值。
        * `version`：Integer，乐观锁版本号。
        * `createdAt`：LocalDateTime，创建时间。
        * `updatedAt`：LocalDateTime，更新时间。
* **错误码**：无特殊业务错误码。
* **备注**：仅返回 `status = ACTIVE` 的模板，不涉及缓存。

### 获取组件模板详情

* **接口类型**：Admin
* **请求方法**：GET
* **接口路径**：`/admin/api/page-builder/component-templates/{code}`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminComponentTemplateController.java#getTemplateDetail`
* **鉴权要求**：需角色：管理员
* **功能说明**：根据组件唯一编码获取单个组件模板的完整详情。
* **请求参数**：
    * `code`：String，路径变量，必填，组件唯一编码。
* **响应字段**：
    * `data`：`ComponentTemplateVO`，组件模板详情（字段同列表）。
* **错误码**：
    * `40401`（`COMMON_RESOURCE_NOT_FOUND`）：指定的组件模板不存在或已被禁用。
* **备注**：仅查询 `status = ACTIVE` 的模板。

### 创建页面

* **接口类型**：Admin
* **请求方法**：POST
* **接口路径**：`/admin/api/page-builder/pages`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminPageDefinitionController.java#createPage`
* **鉴权要求**：需角色：管理员
* **功能说明**：创建新页面定义，校验 pageKey 和 routePath 唯一性，自动在事务内初始化空白草稿记录。
* **请求参数**（`PageDefinitionCreateDTO`）：
    * `pageKey`：String，必填，最长 64 字符，页面唯一 Key 标识。
    * `name`：String，必填，最长 128 字符，页面名称。
    * `routePath`：String，必填，最长 255 字符，页面访问路由路径。
    * `pageType`：String，必填，页面类型（NORMAL / SYSTEM）。
    * `visible`：Boolean，必填，是否前台可见。
    * `sortOrder`：Integer，必填，≥ 0，排序值。
* **响应字段**：`data`：`PageDefinitionVO`，创建成功的页面定义详情。
* **错误码**：
    * `90002`（`PAGE_KEY_DUPLICATE`）：pageKey 已存在（含 DKE 兜底）。
    * `90003`（`PAGE_ROUTE_DUPLICATE`）：routePath 已存在。
    * `40001`（`COMMON_PARAM_INVALID`）：pageType 不合法。
* **备注**：事务内级联创建草稿记录，记录审计日志。

### 更新页面

* **接口类型**：Admin
* **请求方法**：PUT
* **接口路径**：`/admin/api/page-builder/pages/{id}`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminPageDefinitionController.java#updatePage`
* **鉴权要求**：需角色：管理员

* **请求参数**：
    * `id`：Long，路径变量，必填，页面定义 ID。
    * `name`：String，必填，最长 128 字符。
    * `routePath`：String，必填，最长 255 字符。
    * `visible`：Boolean，必填。
    * `sortOrder`：Integer，必填，≥ 0。
    * `version`：Integer，必填，乐观锁版本号。
* **响应字段**：`data`：`Array<PageDefinitionVO>`，更新后的全量页面列表。
* **错误码**：
    * `90001`（`PAGE_NOT_FOUND`）：页面不存在。

    * `90006`（`COMMON_STATE_CONFLICT`）：版本冲突。
* **备注**：记录审计日志（变更前后快照）。

### 保存页面草稿

* **接口类型**：Admin
* **请求方法**：PUT
* **接口路径**：`/admin/api/page-builder/drafts/{pageId}`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminPageDraftController.java#saveDraft`
* **鉴权要求**：需角色：管理员
* **功能说明**：保存/更新页面草稿 Schema。执行乐观锁版本校验 → `PageSchemaValidationService` Schema 合规性校验（白名单、XSS 清洗、链接协议、媒体 ID 格式）→ 计算 SHA-256 哈希 → 持久化。
* **请求参数**（`PageDraftSaveDTO`）：
    * `pageId`：Long，路径变量，必填，页面定义 ID。
    * `schemaJson`：`PageSchemaModel`，必填，页面完整 Schema 配置。
    * `editorSessionRemark`：String，可选，最长 255 字符，编辑备注。
    * `version`：Integer，必填，乐观锁版本号。
* **响应字段**：`data`：`PageDraftVO`，更新后的草稿详情。
* **错误码**：
    * `90004`（`PAGE_DRAFT_NOT_FOUND`）：草稿不存在。
    * `90006`（`COMMON_STATE_CONFLICT`）：版本冲突。
    * `40001`（`COMMON_PARAM_INVALID`）：Schema 校验失败（组件不存在、必填项缺失、链接协议非法、媒体 ID 格式错误等）。
    * `50001`（`SYSTEM_ERROR`）：哈希算法不可用或序列化失败。
* **备注**：记录审计日志（变更前后草稿快照）。

### 生成受控预览 Token（新）

* **接口类型**：Admin
* **请求方法**：POST
* **接口路径**：`/admin/api/page-builder/drafts/{pageId}/previews`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminPreviewController.java#createPreview`
* **鉴权要求**：需角色：管理员
* **功能说明**：为指定页面的当前草稿生成受控预览 Token（随机 UUID），绑定 pageId / schemaHash / 操作人存入 Redis，TTL 1 小时。可选传入 schemaHash 做一致性校验。
* **请求参数**：
    * `pageId`：Long，路径变量，必填，页面定义 ID。
    * 请求体（可选 JSON）：`schemaHash`：String，前端当前草稿哈希，用于服务端比对。
* **响应字段**：
    * `previewToken`：String，随机 UUID，不含业务数据。
    * `previewUrl`：String，完整预览访问地址。
    * `expiresAt`：LocalDateTime，Token 过期时间。
* **错误码**：
    * `90004`（`PAGE_DRAFT_NOT_FOUND`）：草稿不存在或 Schema 为空。
    * `90006`（`PAGE_PREVIEW_SCHEMA_HASH_MISMATCH`）：前端 schemaHash 与服务端不一致，请先保存。
* **备注**：不参与数据库事务，仅操作 Redis；日志中 Token 只打印前 6 位掩码。

## Phase 4 接口

### 发布页面

* **接口类型**：Admin
* **请求方法**：POST
* **接口路径**：`/admin/api/page-builder/pages/{id}/publish`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminPagePublishController.java#publishPage`
* **鉴权要求**：需角色：管理员
* **功能说明**：将指定页面的草稿发布上线。流程：校验草稿 → 生成历史版本（`PUBLISH_BASE`） → 旧 ACTIVE 快照置为 SUPERSEDED → 生成新 ACTIVE 快照 → 提取媒体/实体依赖 → 记录审计日志 → 失效 Portal 缓存（afterCommit）。
* **请求参数**：
    * `id`：Long，路径变量，必填，页面定义 ID。
    * `changeSummary`：String，必填，最长 255 字符，发布变更描述。
* **响应字段**：
    * `data`：`PageVersionVO`，生成的历史版本。
        * `id`：Long，版本 ID。
        * `pageId`：Long，页面定义 ID。
        * `versionNo`：Integer，版本序号。
        * `sourceType`：String，版本来源（PUBLISH_BASE）。
        * `schemaJson`：`PageSchemaModel`，发布时 Schema 快照。
        * `schemaHash`：String，SHA-256 哈希。
        * `changeSummary`：String，变更描述。
        * `version`：Integer，乐观锁版本号。
        * `createdAt`：LocalDateTime，创建时间。
* **错误码**：
    * `90004`（`PAGE_DRAFT_NOT_FOUND`）：草稿不存在或 Schema 为空。
    * `90001`（`PAGE_NOT_FOUND`）：页面不存在。
* **备注**：事务内操作，缓存失效通过 `PortalCacheInvalidationSupport` 在 afterCommit 中执行。

### 回滚页面

* **接口类型**：Admin
* **请求方法**：POST
* **接口路径**：`/admin/api/page-builder/pages/{id}/rollback`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/AdminPagePublishController.java#rollbackPage`
* **鉴权要求**：需角色：管理员
* **功能说明**：将指定页面回滚到指定的历史版本。流程：校验版本 → 生成回滚版本（`ROLLBACK_BASE`） → 旧 ACTIVE 快照置为 SUPERSEDED → 生成新 ACTIVE 快照 → 同步覆盖草稿 → 重建依赖 → 审计日志 → 失效缓存。
* **请求参数**：
    * `id`：Long，路径变量，必填，页面定义 ID。
    * `versionId`：Long，请求参数，必填，目标历史版本 ID。
* **响应字段**：`data`：`PageVersionVO`，新生成的回滚版本（字段同发布）。
* **错误码**：
    * `40401`（`COMMON_RESOURCE_NOT_FOUND`）：指定的历史版本不存在。
    * `90001`（`PAGE_NOT_FOUND`）：页面不存在。
* **备注**：回滚后草稿被同步覆盖为历史版本内容。

## Phase 5 接口

### 获取已发布页面渲染数据

* **接口类型**：Portal
* **请求方法**：GET
* **接口路径**：`/portal/api/page-builder/pages`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/PortalPageController.java#getPageRender`
* **鉴权要求**：无需登录（匿名可访问）
* **功能说明**：根据路由路径获取已发布页面的完整渲染模型。流程：读缓存 → 加载 ACTIVE 快照 → 解析装配数据绑定（跨 11 个业务模块）→ 擦除绑定配置元数据 → 写缓存 → 返回。
* **请求参数**：
    * `routePath`：String，请求参数，必填，页面访问路由路径（如 `/about`）。
* **响应字段**：
    * `data`：`PortalPageVO`。
        * `pageKey`：String，页面唯一 Key 标识。
        * `name`：String，页面名称。
        * `layout`：`LayoutModel`，布局配置。
        * `sections`：`Array<PortalSectionVO>`，区块列表。
            * `id`：String，区块实例 ID。
            * `component`：String，组件编码。
            * `props`：`Map<String, Object>`，组件属性。
            * `style`：`Map<String, Object>`，样式配置。
            * `visible`：Boolean，是否可见。
            * `bindingData`：Object，装配后的业务数据（无绑定配置元数据）。
        * `seo`：`SeoModel`，SEO 信息。
* **错误码**：
    * `90001`（`PAGE_NOT_FOUND`）：页面不存在、未启用、未发布或快照损坏。
* **备注**：基于 Redis 二级缓存（`PortalCacheSupport`），缓存 Key 为 `official:portal:page:{routePath}`；绑定配置在输出前被擦除（`setBinding(null)`）。

### 获取已发布页面元数据

* **接口类型**：Portal
* **请求方法**：GET
* **接口路径**：`/portal/api/page-builder/pages/{pageKey}/meta`
* **实现位置**：`src/main/java/.../modules/pagebuilder/controller/PortalPageController.java#getPageMeta`
* **鉴权要求**：无需登录（匿名可访问）
* **功能说明**：根据页面唯一 Key 获取已发布页面的元数据配置（SEO 和布局信息），不进行具体区块数据的绑定装配。
* **请求参数**：
    * `pageKey`：String，路径变量，必填，页面唯一 Key（如 `about`）。
* **响应字段**：
    * `data`：`PortalPageMetaVO`。
        * `pageKey`：String，页面唯一 Key 标识。
        * `name`：String，页面名称。
        * `layout`：`LayoutModel`，布局配置。
        * `seo`：`SeoModel`，SEO 信息（title, keywords, description）。
* **错误码**：
    * `90001`（`PAGE_NOT_FOUND`）：页面不存在、未启用或未发布。
* **备注**：基于 Redis 二级缓存，缓存 Key 为 `official:portal:page-meta:{pageKey}`。
