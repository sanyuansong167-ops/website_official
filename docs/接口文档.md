## 维护规则

* **接口变更必须更新**：新增接口、修改接口路径、修改请求参数、修改响应字段、修改错误码、删除接口时，必须同步更新本文档。
* **必须标注实现位置**：每个接口条目必须写明 Controller 文件路径和方法名，便于快速定位代码。
* **必须区分接口类型**：后台接口归入 Admin 接口，前台公开接口归入 Portal 接口。
* **必须记录鉴权要求**：每个接口必须说明是否需要登录、需要什么权限、是否可匿名访问。
* **必须记录数据可见性**：Portal 接口必须说明是否过滤隐藏和逻辑删除数据；草稿、审批、定时发布等状态仅在PRD明确引入后记录。
* **必须记录错误码**：接口可能返回的业务错误码必须写入对应条目，错误码按 `docs/错误码说明.md` 中的数字分段规则设置。
* **文档先于交付**：接口代码完成但本文档未更新，视为接口未完成。

## Portal 缓存自动化机制

所有 Portal 公开只读查询接口（`/portal/api/**` 下的 GET 接口）默认纳入 Redis 缓存自动化机制，由 `infrastructure.cache.PortalCacheSupport` 统一承接读取、写入与失效。

### 缓存 Key 命名规则

统一格式：`official:portal:{module}[:{segment}...]`，由 `PortalCacheKeyBuilder` 构造，业务代码禁止手写完整 Key。

### 读取链路（read-through）

1. 按 module/segment 构建缓存 Key
2. 从 Redis 读取；命中则反序列化返回，不再访问数据库
3. 反序列化失败（含 Redis 字节损坏抛 `SerializationException`、或转换为目标 VO 类型失败）视为缓存损坏：**主动删除坏缓存**并回源数据库，实现自愈
4. Redis 连接故障等非数据损坏异常时仅降级查库，不触发删除，不向前台报错
5. 未命中则查库、组装 VO、写回缓存

### 写入与 TTL

- 非空结果使用 `official.cache.default-ttl`（默认 10min，可按模块覆盖）
- 空结果使用 `official.cache.empty-result-ttl`（默认 5min），降低缓存穿透风险
- per-module 覆盖：`official.cache.portal-ttl-overrides.{module}` 可为指定模块单独设置 TTL；`{module}` 取缓存 Key 的**模块段**（`PortalCacheKeyBuilder.build` 的首参），与 Key 命名空间保持一致
  - 单段 Key：模块段即唯一段，如 `ai_cards`、`honors`、`site-config`
  - 多段 Key：模块段为首段，其下所有 segment 共享同一覆盖键，如 `home` 同时作用于 `official:portal:home:banner` 与 `official:portal:home:metrics`
- 所有缓存必须设置 TTL，不允许永久缓存

### 失效链路（after-commit + 延迟二次删除）

Admin 任意 Create/Update/Delete/显示隐藏/排序等写操作成功后：

1. 在数据库事务提交后（`afterCommit`）立即删除对应 Portal Key
2. 调度延迟二次删除（`official.cache.second-delete-delay`，默认 1s）兜底并发读旧值回填
3. 事务回滚时不得删除缓存
4. 缓存删除失败仅记日志，不回滚业务事务，由 TTL 兜底
5. 一次写操作影响多个 Key 时必须全部清除（如聚合接口依赖子模块）

### 失效责任归属

- 缓存失效由触发写操作的业务 Service 显式声明需失效的 Key
- 基础设施层只负责执行删除，不猜测 Key
- 聚合接口依赖关系由 application 层显式维护

### 不纳入缓存的接口

- Portal 表单提交、上传、下载、回调接口
- Admin 任意查询接口
- 登录、鉴权、权限类接口
- 限流计数、验证码等安全控制接口
- 线索后台看板等敏感数据查询

### 1. 全局配置与基础模块

#### 1.1 站点基础配置

### 获取站点基础配置

* **接口类型**：Admin
* **请求方法**：GET
* **接口路径**：`/admin/api/site/config`
* **实现位置**：`src/main/java/com/company/officialwebsite/modules/site/controller/AdminSiteConfigController.java#getConfig`
* **鉴权要求**：需登录；需角色：管理员。
* **功能说明**：返回后台当前可编辑的站点基础配置，用于初始化管理表单。
* **请求参数**：无。
* **响应字段**：
    * `id`：long，站点配置主键。
    * `version`：int，乐观锁版本。
    * `siteTitle`：string，网站标题。
    * `seoKeywords`：string，SEO 关键字。
    * `seoDescription`：string，SEO 描述。
    * `brandSlogan`：string，全局品牌主张。
    * `brandTagline`：string，辅助宣传语。
    * `logoLightMediaId`：long，亮色 Logo 资源 ID，可为空。
    * `logoLightUrl`：string，亮色 Logo 访问地址，可为空。
    * `logoDarkMediaId`：long，暗色 Logo 资源 ID，可为空。
    * `logoDarkUrl`：string，暗色 Logo 访问地址，可为空。
    * `updatedAt`：string，最近更新时间。
* **错误码**：
    * `20001`：未登录访问。
    * `20002`：非管理员访问。
    * `10002`：默认站点配置记录不存在。
* **备注**：站点配置为单例记录，接口只返回活跃配置。

### 更新站点基础配置

* **接口类型**：Admin
* **请求方法**：PUT
* **接口路径**：`/admin/api/site/config`
* **实现位置**：`src/main/java/com/company/officialwebsite/modules/site/controller/AdminSiteConfigController.java#updateConfig`
* **鉴权要求**：需登录；需角色：管理员；必须携带有效 CSRF Token。
* **功能说明**：更新网站 Title、SEO 信息、品牌文案及亮暗双 Logo。
* **请求参数**：
    * `version`：int，必填，当前配置版本号。
    * `siteTitle`：string，必填，最长 120 字符。
    * `seoKeywords`：string，选填，最长 255 字符。
    * `seoDescription`：string，选填，最长 500 字符。
    * `brandSlogan`：string，必填，最长 160 字符。
    * `brandTagline`：string，选填，最长 255 字符。
    * `logoLightMediaId`：long，选填，亮色 Logo 资源 ID。
    * `logoDarkMediaId`：long，选填，暗色 Logo 资源 ID。
* **响应字段**：同“获取站点基础配置”。
* **错误码**：
    * `10001`：请求参数缺失、长度不合法。
    * `10002`：站点配置或引用的媒体资源不存在。
    * `10003`：配置版本冲突，需要刷新后重试。
    * `20001`：未登录访问。
    * `20002`：非管理员访问。
    * `20005`：缺少 CSRF Token 或 Token 无效。
    * `30001`：Logo 引用的媒体不是可公开图片资源。
* **备注**：保存成功后写入审计日志，并在事务提交后失效 `official:portal:site-config` 缓存。

#### 1.2 导航菜单管理

### 获取导航菜单树

* **接口类型**：Admin
* **请求方法**：GET
* **接口路径**：`/admin/api/site/navigation/menus`
* **实现位置**：`src/main/java/com/company/officialwebsite/modules/site/controller/AdminNavigationMenuController.java#getMenuTree`
* **鉴权要求**：需登录；需角色：管理员。
* **功能说明**：返回后台可编辑的完整顶部导航树，包含隐藏菜单与二级菜单结构。
* **请求参数**：无。
* **响应字段**：
    * `id`：long，菜单 ID。
    * `parentId`：long，父菜单 ID，一级菜单固定为 `0`。
    * `menuLevel`：string，菜单层级，取值 `LEVEL_1` / `LEVEL_2`。
    * `menuName`：string，菜单名称。
    * `targetType`：string，跳转类型，取值 `GROUP` / `INTERNAL_ROUTE` / `PAGE_ANCHOR` / `EXTERNAL_LINK`。
    * `routePath`：string，内部路由，仅 `INTERNAL_ROUTE` 时返回。
    * `anchorCode`：string，页面锚点，仅 `PAGE_ANCHOR` 时返回。
    * `externalUrl`：string，外部链接，仅 `EXTERNAL_LINK` 时返回。
    * `openInNewTab`：boolean，外部链接是否新窗口打开。
    * `visible`：boolean，菜单是否显示。
    * `sortOrder`：int，当前排序值。
    * `version`：int，乐观锁版本。
    * `updatedAt`：string，最近更新时间。
    * `children`：array，二级菜单列表。
* **错误码**：
    * `20001`：未登录访问。
    * `20002`：非管理员访问。
* **备注**：接口返回活跃菜单树，不返回逻辑删除记录。

### 新增导航菜单

* **接口类型**：Admin
* **请求方法**：POST
* **接口路径**：`/admin/api/site/navigation/menus`
* **实现位置**：`src/main/java/com/company/officialwebsite/modules/site/controller/AdminNavigationMenuController.java#createMenu`
* **鉴权要求**：需登录；需角色：管理员；必须携带有效 CSRF Token。
* **功能说明**：新增一级或二级导航菜单，并返回最新后台导航树。
* **请求参数**：
    * `parentId`：long，必填，父菜单 ID；一级菜单传 `0`。
    * `menuName`：string，必填，最长 64 字符。
    * `targetType`：string，必填，取值 `GROUP` / `INTERNAL_ROUTE` / `PAGE_ANCHOR` / `EXTERNAL_LINK`。
    * `routePath`：string，选填，内部路由。
    * `anchorCode`：string，选填，页面锚点，不带 `#`。
    * `externalUrl`：string，选填，外部链接，仅允许 `http/https`。
    * `openInNewTab`：boolean，必填，仅 `EXTERNAL_LINK` 时可为 `true`。
    * `visible`：boolean，必填，是否显示。
* **响应字段**：同“获取导航菜单树”。
* **错误码**：
    * `10001`：请求参数缺失、长度不合法。
    * `10002`：父菜单不存在。
    * `20001`：未登录访问。
    * `20002`：非管理员访问。
    * `20005`：缺少 CSRF Token 或 Token 无效。
    * `30002`：跳转目标配置不合法。
    * `30003`：菜单层级不合法。
    * `30004`：父菜单不存在或不可挂接。
    * `30005`：同级菜单名称重复。
* **备注**：新增菜单默认追加到同级末尾；二级菜单只能挂在一级菜单下。

    * `routePath`：string，选填，内部路由。

    * `externalUrl`：string，选填，外部链接。
    * `openInNewTab`：boolean，必填。
    * `visible`：boolean，必填。
* **响应字段**：同“获取导航菜单树”。
* **错误码**：
    * `10001`：请求参数缺失、长度不合法。
    * `10002`：菜单不存在。

### 删除导航菜单

    * `previewUrl`：string，拼接好的完整前端预览访问地址（由 `official.page-builder.preview-url-template` 配置）。
    * `expiresAt`：datetime，Token 过期时间（UTC+8）。
* **错误码**：
    * `90004`：草稿不存在或草稿 Schema 为空（未编辑，不允许预览）。
    * `90006`：前端 schemaHash 与服务端不一致，请先保存草稿。

### 访问受控预览（新）
* **接口类型**：Portal
* **请求方法**：GET
* **接口路径**：`/portal/api/page-builder/previews/{previewToken}`
* **实现位置**：`src/main/java/com/company/officialwebsite/modules/pagebuilder/controller/PortalPreviewController.java#getPreview`
* **鉴权要求**：无需登录（通过 Token 保证合法性）。
* **功能说明**：通过受控预览 Token 渲染草稿页面，复用与正式发布完全一致的数据绑定装配和可见性过滤，清除所有 binding 元数据后返回渲染结果。预览结果不写入正式 Portal 缓存。
* **响应 Header**：`Cache-Control: no-store`（禁止任何中间代理缓存预览响应）。
* **请求参数**：
    * `previewToken`：string，路径参数，UUID 格式的预览令牌。
* **响应字段**：同 Portal 已发布页面渲染接口的 `PortalPageVO` 结构（pageKey / name / layout / seo / sections），sections 中**不含任何 binding 字段**。
* **错误码**：
    * `90005`：预览 Token 不存在、已过期或已被撤销，统一返回 401 HTTP 状态，不区分具体原因。
* **安全约束**：
    * 响应中禁止出现 `binding`、内部模块名、查询配置等后台内部字段。
    * 绑定的隐藏或逻辑删除实体不出现在响应中（与正式 Portal 行为一致）。

* **接口路径**：`/admin/api/page-builder/previews/{previewToken}`

* **请求参数**：

* **响应字段**：无（data 为 null）。

### Portal 按路由渲染页面
* **接口类型**：Portal
* **请求方法**：GET
* **接口路径**：`/portal/api/page-builder/pages`
* **实现位置**：`src/main/java/com/company/officialwebsite/modules/pagebuilder/controller/PortalPageController.java#getPageRender`
* **鉴权要求**：无需登录，可匿名访问。
* **功能说明**：根据路由路径获取该页面最新发布快照的完整渲染数据，包含各 Section 的数据绑定解析结果，用于前台 SSR/CSR 渲染。
* **请求参数**：
    * `routePath`：string，查询参数，必填，页面路由路径，如 `/home`。
* **响应字段**：
    * `pageKey`：string，页面唯一 Key。
    * `name`：string，页面名称。
    * `seo`：object，SEO 配置，含 `title`、`keywords`、`description`。
    * `layout`：object，布局配置，含 `type`。
    * `sections`：array，渲染后的区块列表，每项包含：

        * `component`：string，组件编码。
        * `props`：object，静态属性配置。

* **错误码**：
    * `90001`：页面不存在或尚未发布。
* **备注**：结果通过 Redis 缓存（Key：`official:portal:page:{routePath}`），Admin 侧任何涉及该页面依赖实体的写操作都会在事务提交后触发联动失效；仅返回 visible=true 的 Section；binding 绑定配置字段不对外输出。

# Release Baseline v1.0 接口契约补充

> 本节为发布前收口补充，覆盖本轮代码中已经存在但旧版接口文档未完整展开的接口。历史接口说明不删除，本节作为 v1.0 发布基线的增量事实源。

## 通用规则

- Admin 接口：需要 Cookie Session；除登录、CSRF 获取外，写操作必须携带 CSRF Token。
- Portal 接口：匿名可访问；写接口按具体说明执行限流与校验。
- 通用错误码：`10001` 请求参数错误；`20001` 未登录；`20002` 无权限；`30001` 数据不存在；`30002` 乐观锁版本冲突；`80000` 系统内部异常。
- 通用字段：`id` 为 long；`version` 为 int；`visible` 为 boolean；`status` 为 string；`sortOrder` 为 int；`createdAt/updatedAt` 为 ISO-8601 时间字符串。

# Release Baseline v1.0 接口契约补充

> 本节为发布前收口补充，覆盖本轮代码中已经存在但旧版接口文档未完整展开的接口。历史接口说明不删除，本节作为 v1.0 发布基线的增量事实源。

## 通用规则

- Admin 接口：需要 Cookie Session；除登录、CSRF 获取外，写操作必须携带 CSRF Token。
- Portal 接口：匿名可访问；写接口按具体说明执行限流与校验。
- 通用错误码：`10001` 请求参数错误；`20001` 未登录；`20002` 无权限；`30001` 数据不存在；`30002` 乐观锁版本冲突；`80000` 系统内部异常。
- 通用字段：`id` 为 long；`version` 为 int；`visible` 为 boolean；`status` 为 string；`sortOrder` 为 int；`createdAt/updatedAt` 为 ISO-8601 时间字符串。
