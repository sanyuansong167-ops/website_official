# 阶段 04：独立内容详情编辑器

> 前置阶段：[阶段 03：全站页面与公共能力实施](./03-全站页面与公共能力实施.md)  
> 后续阶段：[阶段 05：发布治理与上线验收](./05-发布治理与上线验收.md)  
> 阶段目标：交付产品、案例、行业方案的独立详情编辑器，并使详情内容、页面区块、Portal 列表和详情页共享唯一可信内容来源。

## 1. 本阶段范围

本阶段将产品、案例、行业方案从页面区块中的“展示引用”扩展为可独立维护的内容实体。每个实体都必须具备草稿、预览、发布、版本、回滚、媒体引用、SEO 和独占编辑锁。

包含：

1. 产品详情编辑器。
2. 案例详情编辑器。
3. 行业方案详情编辑器。
4. 三类实体的草稿、发布快照、版本历史与回滚。
5. 富文本和媒体引用治理。
6. 内容关联与页面区块绑定。
7. Portal 详情页、列表页和缓存失效联动。

不包含：

- 多人并发编辑或内容合并。
- 复杂审批流、定时发布和多级审核。
- 自定义脚本类型的详情模板。

## 3. 共同数据与状态规则

### 3.1 内容状态

每类内容至少区分：

```text
DRAFT      草稿，仅 Admin 可读取
PUBLISHED  已发布，可按可见性规则在 Portal 展示
OFFLINE    已下线，Portal 不展示
```

规则：

- 草稿更新不得覆盖已发布快照。
- Portal 列表、详情、自动数据绑定仅使用 `PUBLISHED` 且可见、未逻辑删除的数据。
- 下线或删除已被页面引用的内容时，必须明确阻止、自动移除引用或将页面置为待修复状态；具体策略应统一，不能按模块随意处理。

### 3.2 统一详情编辑器框架

三类编辑器复用以下骨架：

```text
顶部：返回列表 | 保存草稿 | 预览 | 发布 | 历史版本
主体：基础信息 | 正文内容 | 媒体资源 | 关联内容 | SEO | 发布设置
侧栏：编辑锁状态 | 最近保存时间 | 版本号 | 变更说明
```

共同要求：

- 进入编辑器时先获取对应实体的独占锁。
- 所有保存与发布请求都携带版本号，并由服务端二次校验锁归属。
- 未保存修改离开页面前必须提示。
- 列表页只显示必要摘要，完整编辑只能进入独立详情页。

## 4. 实施顺序

### 4.1 建立可复用的内容编辑基础

1. 抽取 `ContentEditorShell`，统一管理锁、保存、预览、发布、版本与离开保护。
2. 抽取基础字段、SEO、媒体选择、富文本、关联内容、发布设置等通用面板。
3. 定义各实体的字段 Schema、必填规则、最大长度、枚举值和默认值。
4. 为内容实体实现与页面相同的锁能力：创建锁、心跳、主动释放、超时释放、强制解锁。
5. 定义草稿与发布快照结构，避免直接以线上实体表覆盖历史数据。
6. 为每类资源明确缓存依赖和审计事件。

**交付物**：内容编辑器通用框架、三类实体 Schema、内容编辑锁实现、共用版本与预览能力。

### 4.2 实现产品详情编辑器

地址：`/admin/content/products/{id}`。

按以下顺序实现：

1. 基础信息：产品名称、短标题、摘要、状态、排序、可见性。
2. 媒体信息：Logo、封面、头图、图集与替代文本。
3. 详情内容：富文本正文、能力亮点、适用行业、功能描述。
4. 关联内容：关联案例、关联行业方案、推荐产品。
5. SEO：标题、关键词、描述、默认回退规则。
6. 草稿保存、预览、发布、版本历史、回滚。
7. Portal `/products/{id}` 读取已发布快照，列表页摘要与详情页内容保持一致。

发布校验至少包括：产品名称、摘要、状态、SEO 字段安全性、封面媒体有效性、关联 ID 有效性、富文本安全性。

### 4.3 实现案例详情编辑器

地址：`/admin/content/cases/{id}`。

按以下顺序实现：

1. 基础信息：标题、客户名称、行业、摘要、状态、排序、可见性。
2. 内容结构：项目背景、解决方案、项目成果、详情正文。
3. 媒体信息：封面、图片集、替代文本和媒体引用清单。
4. 关联内容：关联产品、推荐案例、关联行业方案。
5. SEO：标题、关键词、描述。
6. 草稿、预览、发布、版本和回滚。
7. Portal `/cases/{id}` 只输出已发布案例及允许公开的关联内容。

发布校验至少包括：标题、必要业务摘要、富文本安全性、图片可用性、关联内容发布状态和 SEO 安全性。

### 4.4 实现行业方案详情编辑器

地址：`/admin/content/solutions/{id}`。

按以下顺序实现：

1. 基础信息：方案名称、摘要、行业标签、状态、排序、可见性。
2. 方案内容：行业痛点、解决思路、核心能力、应用场景、典型客户。
3. 媒体信息：图标、封面、图集、替代文本和引用清单。
4. 关联内容：关联产品、关联案例、推荐方案。
5. SEO：标题、关键词和描述。
6. 草稿、预览、发布、版本和回滚。
7. 补齐或确认 Portal `/solutions/{id}` 详情接口，只返回已发布方案。

发布校验至少包括：名称、摘要、图标或封面规则、正文安全性、关联内容状态、SEO 和 URL 合法性。

### 4.5 富文本与媒体引用治理

1. 富文本编辑器输出进入服务端前进行结构化净化，禁止脚本、危险事件属性和不允许的协议。
2. 保存详情时提交媒体 ID 引用清单；服务端维护引用关系，不以 HTML 扫描作为唯一依据。
3. 删除或下线媒体前检查引用；被发布内容引用的媒体不得直接删除。
4. 图片、PDF 等引用资源必须按公开权限、有效状态和媒体类型校验。
5. 详情预览和 Portal 输出使用同一套媒体 URL 生成与权限控制规则。

### 4.6 关联关系与页面绑定

1. 页面区块只保存实体 ID、排序和展示配置，不复制详情正文。
2. 详情编辑器显示“被哪些页面/区块引用”，便于发布前评估影响范围。
3. 发布产品、案例或方案后，按引用关系失效相关页面、列表和详情缓存。
4. 下线、删除或回滚内容时，对引用页面执行预定义处理：阻止操作、自动隐藏引用或提示管理员修复；选定规则后全模块一致执行。
5. 关联产品、案例、方案只能引用符合当前业务规则的已发布或允许预览实体。

## 5. 建议接口能力

实际路径以项目接口文档为准；本阶段必须为三类实体补齐等价能力：

| 能力 | 产品示例 | 案例/方案要求 |
| --- | --- | --- |
| 获取编辑草稿 | `GET /admin/api/products/{id}/draft` | 采用对应资源路径。 |
| 保存草稿 | `PUT /admin/api/products/{id}/draft` | 校验 CSRF、锁、版本、字段与媒体。 |
| 创建编辑锁 | `POST /admin/api/products/{id}/lock` | 同产品锁协议。 |
| 心跳/释放锁 | `POST/DELETE /admin/api/products/{id}/lock` | 同产品锁协议。 |
| 创建预览 | `POST /admin/api/products/{id}/draft/previews` | Token 受控、可撤销。 |
| 发布 | `POST /admin/api/products/{id}/publish` | 生成发布快照、失效缓存。 |
| 版本/回滚 | `GET /.../versions`、`POST /.../rollback/{versionId}` | 不可变历史版本。 |

所有接口必须声明鉴权、请求字段、响应字段、错误码、缓存影响和 Controller 实现位置。
