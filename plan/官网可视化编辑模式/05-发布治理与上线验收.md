# 阶段 05：发布治理与上线验收

> 前置阶段：[阶段 04：独立内容详情编辑器](./04-独立内容详情编辑器.md)  
> 阶段目标：将页面与独立内容编辑能力提升为可安全上线、可审计、可监控、可回滚的生产级发布系统。

## 1. 本阶段范围

本阶段不扩展新的业务组件，重点完成发布治理、缓存可靠性、测试、可观测性、文档和生产上线准备。

包含：

1. 发布前全量校验与发布失败恢复。
2. 审计日志、错误处理、监控与告警。
3. Portal 缓存失效、延迟二次删除与重建验证。
4. 权限、CSRF、XSS、草稿隔离、预览 Token、锁冲突等安全测试。
5. 单元、接口、E2E、视觉回归和性能测试。
6. 文档同步、预发布演练和生产上线准入。

## 2. 开始条件

- [ ] 阶段 04 退出条件全部通过。
- [ ] 首页、产品、案例、行业方案均具备草稿、预览、发布、回滚和锁能力。
- [ ] 所有发布相关缓存 Key、实体依赖和页面引用关系已登记。
- [ ] 预发布环境具备独立的数据库、Redis、对象存储、日志和监控。
- [ ] 已确定上线负责人、回滚负责人和紧急联系人。

## 3. 实施顺序

### 3.1 固化发布前校验清单

发布请求在服务端必须按以下顺序校验：

1. 请求者已登录、拥有管理员发布权限、CSRF 合法。
2. 请求者持有当前资源的有效编辑锁。
3. 草稿版本与服务端版本一致，不存在冲突。
4. 页面或实体状态允许发布。
5. Schema、组件、插槽、排序、可见性和必填字段合法。
6. 富文本已净化，外部链接协议合法，媒体资源存在且可公开展示。
7. 关联产品、案例、方案和页面依赖满足发布规则。
8. SEO 标题、描述和关键词符合长度、内容与安全规则。
9. 发布快照可以序列化、计算哈希并持久化。

任一步失败均不得更新已发布内容、不得失效 Portal 缓存、不得删除原发布快照。

### 3.2 完成事务、快照与缓存一致性

1. 发布、回滚、下线和影响已发布内容的写操作必须在事务提交后触发缓存失效。
2. 使用统一的缓存 Key 构建器，业务代码不得手写完整 Key。
3. 对每次变更明确列出受影响缓存：页面渲染、页面 SEO、首页聚合、内容列表、内容详情、导航或站点配置。
4. 提交后立即删除缓存，并安排延迟二次删除；事务回滚时不得删除缓存。
5. Redis 读取损坏时主动删除坏缓存并回源；Redis 连接异常时降级查库，不向 Portal 暴露内部错误。
6. 缓存删除、延迟任务或重建失败时记录模块、对象、缓存标识、结果、错误摘要和 `traceId`，并进入可重试队列或运维告警。
7. 发布后验证 Portal 从已发布快照读取，而不是意外读取草稿或实体中间状态。

### 3.3 完成审计与可观测性

必须审计：

```text
编辑锁获取、续期、释放、强制解锁
草稿保存
预览 Token 创建与撤销
页面/内容发布、下线与回滚
媒体上传、替换、删除与引用变更
缓存手工刷新和缓存失效失败重试
```

审计最少记录：操作人 ID/名称、资源类型/ID、动作、结果、IP、User-Agent、时间、`traceId`、变更前后核心摘要。

需要建立的监控指标：

- 发布成功率、失败率、平均耗时。
- 锁冲突次数、锁超时次数、强制解锁次数。
- 草稿保存失败率和版本冲突次数。
- 预览 Token 创建、失效和非法访问次数。
- 缓存删除失败、延迟二次删除失败、缓存重建失败次数。
- Portal 页面错误率、接口延迟、LCP/CLS/INP 等前端性能指标。

### 3.4 安全测试

至少验证：

1. 匿名用户与非管理员用户不能访问 Admin 页面、草稿、预览管理、版本和锁接口。
2. 所有 Admin 写接口拒绝缺少或无效 CSRF 的请求。
3. Portal API 不返回草稿、锁信息、审计信息、内部 ID 以外的敏感内部字段或完整 Token。
4. 预览 Token 到期、撤销、篡改、跨资源使用和非授权访问均被拒绝。
5. 富文本中的脚本、危险 URL、事件属性和非法嵌入内容被净化或拒绝。
6. 外部链接、上传媒体和文件类型按白名单处理。
7. 锁归属错误、锁过期和强制解锁后，旧会话无法继续保存或发布。
8. 统一错误响应不泄露 SQL、堆栈、文件路径、密钥、Cookie、Token 或敏感线索内容。

### 3.5 自动化测试与视觉回归

**单元测试**：

- Schema 校验、插槽规则、组件注册表、链接白名单、媒体引用、SEO 校验。
- 锁状态机、超时、心跳、版本冲突和发布状态机。
- 缓存依赖计算和错误归一化。

**接口测试**：

- 草稿保存、预览、发布、回滚、锁接口和所有失败错误码。
- 权限、CSRF、锁冲突、版本冲突、非法 Schema、无效媒体和依赖冲突。
- 缓存失效与 Portal 读取已发布快照。

**E2E 测试**：

- 管理员编辑首页并发布，普通用户验证新内容。
- 两名管理员竞争同一编辑锁。
- 产品、案例、方案详情从草稿到发布、再回滚。
- 预览 Token 失效与撤销。
- 移动端普通官网浏览与桌面端编辑器操作。

**视觉回归**：

- 首页、产品详情、案例详情、行业方案详情的已发布页面。
- `375px`、`768px`、`1024px`、`1440px` 的 Portal 页面截图。
- 编辑器以桌面宽度为准，至少覆盖常用 `1280px` 和 `1440px` 宽度。

### 3.6 文档与发布演练

1. 所有新增/修改接口同步更新 `docs/接口文档.md` 和 `docs/重构接口文档.md`。
2. 所有错误码同步更新 `docs/错误码说明.md`。
3. 表结构、迁移、逻辑删除、审计、媒体引用和缓存记录同步更新 `docs/数据库设计说明.md`。
4. 前端路由、组件、状态、无障碍、性能和 Portal 数据边界同步更新 `docs/官网前端实现说明.md`。
5. 在预发布环境演练：首页发布、产品发布、案例回滚、行业方案下线、缓存故障降级和锁异常释放。
6. 明确生产发布窗口、变更清单、数据备份策略、观察时长、回滚步骤与负责人。

## 4. 交付物

- 发布前全量校验服务与错误提示规范。
- 缓存依赖清单、缓存失效日志与重试/告警能力。
- 可查询的审计日志和发布操作追踪。
- 安全、接口、E2E、视觉回归与性能测试报告。
- 更新后的接口、错误码、数据库、前端和运维文档。
- 预发布演练记录、生产发布方案与回滚预案。

## 5. 验收清单

### 发布与回滚

- [ ] 任意发布前校验失败都不会影响当前 Portal 已发布内容。
- [ ] 发布成功后生成不可变快照、审计记录和正确版本号。
- [ ] 回滚生成新版本，不删除历史版本。
- [ ] 发布和回滚后页面、SEO、列表和详情缓存均正确失效。

### 缓存与可靠性

- [ ] 事务回滚时不发生缓存删除。
- [ ] 事务提交后执行立即删除与延迟二次删除。
- [ ] Redis 故障时 Portal 降级查库，不暴露内部异常。
- [ ] 缓存损坏时可自愈；失效失败可追踪、可告警。

### 安全与权限

- [ ] 所有越权、无 CSRF、无锁、过期锁和版本冲突场景均被拒绝并返回稳定错误码。
- [ ] 草稿、预览、Token、锁、审计和敏感内容不存在匿名泄露路径。
- [ ] 富文本、链接和媒体安全测试通过。

### 体验与质量

- [ ] Portal 首屏未加载编辑器代码或 Admin 请求。
- [ ] 普通官网在指定响应式宽度下无横向溢出、重叠或不可点击控件。
- [ ] 编辑器桌面端关键流程稳定，保存或发布失败时草稿不丢失。
- [ ] 自动化测试通过，关键页面视觉回归无未确认差异。

## 6. 上线退出条件

以下条件必须全部满足：

- [ ] 总计划第 12 章的功能、并发、安全、性能和文档验收项全部通过。
- [ ] 页面编辑器及产品、案例、行业方案详情编辑器完成 P0 验收。
- [ ] 独占编辑锁、异常超时释放和强制解锁完成预发布验证。
- [ ] 发布、回滚和缓存失效完成端到端演练。
- [ ] 审计日志、监控指标、告警渠道和异常处置责任人已确认。
- [ ] 数据库迁移可在空库执行，风险变更具有回滚或兼容说明。
- [ ] 已完成生产发布窗口审批、变更公告、备份确认和回滚演练。
