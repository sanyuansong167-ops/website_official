# 阶段 01：编辑器基础设施与独占锁

> 前置阶段：无  
> 后续阶段：[阶段 02：首页可视化编辑](./02-首页可视化编辑.md)  
> 阶段目标：建立安全、可扩展的可视化编辑基础，确保只有管理员能编辑草稿，且同一资源不可被多人同时修改。

## 1. 本阶段范围

本阶段只交付编辑器“骨架能力”，不要求完成全部首页区块或三类内容详情编辑器。

包含：

1. 管理员编辑器路由、鉴权与前端访问隔离。
2. 官网组件的 `view`、`preview`、`edit` 三种渲染模式。
3. 页面定义、页面草稿读取与保存。
4. 受控组件注册表、插槽规则与区块 Schema 校验。
5. 页面独占编辑锁、心跳、自动释放、主动释放与强制解锁。
6. 基础审计、统一错误处理、版本校验和测试骨架。

不包含：

- 全部首页区块的完整编辑表单。
- 产品、案例、行业方案独立详情编辑器。
- 移动端管理员编辑器。
- 多人协作、实时协同或编辑内容合并。

## 2. 开始条件

- [ ] Admin 登录态、角色权限和 CSRF 基础设施可用。
- [ ] Portal 与 Admin API 已按文档分离。
- [ ] 页面构建器基础数据模型可用，至少可读取页面定义、草稿和发布快照。
- [ ] 已明确管理员角色标识为 `ADMINISTRATOR`，并确认更高权限账号的强制解锁授权规则。
- [ ] 已确定首个试点页面为首页 `/`，并拥有可用于开发与测试的页面定义数据。

## 3. 实施顺序

### 3.1 建立编辑器路由与安全边界

1. 创建受保护的编辑器入口：`/admin/editor?routePath={path}`。
2. 在路由加载前校验登录态与 `ADMINISTRATOR` 角色；不满足时跳转登录页或返回无权限页。
3. 禁止在 Portal 公共路由渲染“编辑官网”按钮、编辑器 JavaScript 包、草稿请求或 Admin API 请求。
4. 在 Admin 后台增加“可视化编辑官网”入口，仅对授权管理员展示。
5. 定义非法 `routePath` 处理：必须以 `/` 开头，禁止访问非页面构建器路由、管理路由或协议地址。

**交付物**：受保护路由、Admin 入口、无权限页、路由参数校验。

### 3.2 抽取共享页面渲染器

1. 将 Header、Footer、容器、Section 容器和页面渲染器从具体路由逻辑中抽离。
2. 定义统一渲染上下文：`mode`、页面 Schema、动态数据、当前选中区块、编辑能力回调。
3. 为每个区块组件规定三种模式的最小行为：
   - `view`：仅展示已发布内容。
   - `preview`：展示草稿但无编辑控件。
   - `edit`：展示草稿，并允许编辑器外层注入选中、工具栏与拖拽能力。
4. 建立未知组件降级策略：编辑器显示“未注册组件”错误；Portal 跳过该区块并记录可观测日志。
5. 确保 `view` 组件不依赖编辑器状态、Admin DTO 或草稿字段。

**交付物**：`PageRenderer`、`EditableSection`、模式类型定义、未知组件降级策略。

### 3.3 定义受控组件与插槽 Schema

1. 建立组件注册表，首期至少注册 `HeroSection`、`RichTextSection`、`ImageTextSection`、`ProductGridSection`、`CaseGridSection`、`SolutionGridSection`、`ContactCtaSection` 等类型。
2. 为每个组件定义：组件编码、允许插槽、可重复性、必填字段、默认配置、允许的数据绑定、受控样式字段。
3. 建立页面插槽规则：`header`、`main`、`footer`；仅 `main` 首期允许自由增加区块。
4. 规定全局 Header、Footer 不可被普通编辑操作删除；需要变更时使用独立受控配置。
5. 在服务端实现 Schema 校验：组件白名单、插槽合法性、必填字段、链接协议、媒体 ID、排序值、富文本安全规则。
6. 在前端实现同等基础校验以改善体验，但以后端校验结果为唯一准则。

**交付物**：组件注册表、JSON Schema 或等价校验模型、插槽规则、默认区块配置。

### 3.4 接入页面草稿读写

1. 使用现有页面构建器接口读取页面定义和当前草稿。
2. 编辑器加载顺序为：校验权限 → 创建编辑锁 → 读取页面定义 → 读取草稿 → 初始化画布。
3. 未创建草稿时，按当前发布快照或受控默认 Schema 初始化草稿，禁止空白覆盖已发布内容。
4. 保存草稿时必须提交页面 ID、完整 Schema、草稿版本号和可选变更说明。
5. 后端执行乐观锁、Schema 校验、XSS 清洗、哈希计算、草稿持久化和审计写入。
6. 保存成功后更新前端草稿版本和哈希；保存失败不得清空用户当前画布。

**交付物**：页面草稿查询/保存适配层、草稿状态管理、保存状态与失败恢复交互。

### 3.5 实现独占编辑锁

1. 定义锁资源键：`resourceType + resourceId`；页面首期使用 `PAGE + pageId`。
2. 创建加锁接口，例如 `POST /admin/api/page-builder/pages/{pageId}/lock`。
3. 加锁成功返回锁 ID、持有人、过期时间与心跳建议间隔；已被他人持有时返回稳定业务错误码和脱敏后的占用提示。
4. 编辑器在锁有效期内定时调用心跳接口；心跳失败应提示管理员保存风险并阻止继续提交发布操作。
5. 实现主动解锁：退出编辑器、取消编辑、发布成功后释放锁。
6. 实现自动释放：锁在未收到心跳后超时失效，不依赖浏览器关闭事件。
7. 实现强制解锁：仅更高权限管理员可用，必须填写原因，写入审计日志，并通知被解锁会话其锁已失效。
8. 所有保存、预览、发布、回滚接口都必须再次验证当前请求者持有有效锁。

**交付物**：编辑锁数据模型或 Redis 锁实现、加锁/心跳/解锁/强制解锁接口、锁冲突前端页面、审计事件。

### 3.6 统一异常、审计与测试骨架

1. 统一处理未登录、无权限、锁冲突、版本冲突、草稿不存在、Schema 非法和系统异常。
2. 错误码遵循 `docs/错误码说明.md`，业务代码不允许使用裸数字。
3. 为以下行为写审计日志：获取编辑锁、保存草稿、释放锁、强制解锁、预览、发布、回滚。
4. 日志记录操作人、资源类型、资源 ID、操作结果、IP、User-Agent、`traceId` 和必要的变更摘要；不得记录完整预览 Token 或敏感内容。
5. 建立单元测试、接口测试与 E2E 测试目录和基础夹具。

## 4. 建议接口清单

| 能力 | 方法与路径 | 必须校验 |
| --- | --- | --- |
| 读取草稿 | `GET /admin/api/page-builder/drafts/{pageId}` | 管理员权限。 |
| 保存草稿 | `PUT /admin/api/page-builder/drafts/{pageId}` | 管理员、CSRF、有效锁、版本、Schema。 |
| 创建锁 | `POST /admin/api/page-builder/pages/{pageId}/lock` | 管理员。 |
| 锁心跳 | `POST /admin/api/page-builder/pages/{pageId}/lock/heartbeat` | 管理员、锁归属。 |
| 主动解锁 | `DELETE /admin/api/page-builder/pages/{pageId}/lock` | 管理员、锁归属。 |
| 强制解锁 | `POST /admin/api/page-builder/pages/{pageId}/lock/force-release` | 更高权限、CSRF、解锁原因。 |

接口落地后必须补充到 `docs/接口文档.md` 与 `docs/重构接口文档.md`，并记录实际 Controller 位置和错误码。

## 5. 测试与验收

### 必测场景

- [ ] 未登录用户访问 `/admin/editor` 被拒绝。
- [ ] 非管理员登录用户访问、保存、加锁和解锁均被拒绝。
- [ ] Portal 页面不请求任何 Admin API，也不包含编辑工具栏。
- [ ] 管理员 A 成功获取首页编辑锁。
- [ ] 管理员 B 访问同一首页时只能查看锁定提示，不能保存草稿。
- [ ] 管理员 A 发送心跳后锁持续有效；心跳停止后锁按配置超时释放。
- [ ] 管理员 A 退出编辑器后锁被主动释放。
- [ ] 强制解锁只有授权账号可执行，且产生审计日志。
- [ ] 无有效锁、过期锁或错误锁归属时保存草稿被拒绝。
- [ ] 非法组件、非法插槽、脚本链接、无效媒体 ID 和缺失必填字段无法保存。
- [ ] 保存版本冲突时不覆盖原草稿，并向编辑器返回可行动提示。

### 阶段退出条件

- [ ] 首页 `/` 可由管理员在编辑器中加载草稿并保存合法 Schema。
- [ ] 编辑器与 Portal 使用同一渲染器，至少一个示例区块在 `view`、`preview`、`edit` 三种模式下效果正确。
- [ ] 页面编辑锁覆盖创建、心跳、主动释放、超时释放和强制解锁。
- [ ] 无越权访问、草稿泄露或静默覆盖问题。
- [ ] 接口、错误码、数据库迁移和审计说明已同步文档化。
