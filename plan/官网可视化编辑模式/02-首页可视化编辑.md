# 阶段 02：首页可视化编辑

> 前置阶段：[阶段 01：编辑器基础设施与独占锁](./01-编辑器基础设施与独占锁.md)  
> 后续阶段：[阶段 03：全站页面与公共能力实施](./03-全站页面与公共能力实施.md)  
> 阶段目标：以首页为试点，完成管理员从草稿编辑、受控预览到正式发布与回滚的完整可视化交付链路。

## 1. 本阶段范围

本阶段将首页作为第一个完整可发布页面，验证受控区块、媒体管理、数据绑定、发布快照、缓存失效和回滚的协同关系。

首期首页区块：

```text
HeroSection
MetricsSection
ImageTextSection / StrengthSummarySection
AiCardSection
CapabilitySection
ProductGridSection
SolutionGridSection
CaseGridSection
ContactCtaSection
```

不要求首页首期支持所有未来组件；但新增区块的机制必须可复用，不可为每个区块编写独立页面逻辑。

## 2. 开始条件

- [ ] 阶段 01 退出条件全部通过。
- [ ] 首页具有页面定义、已发布快照和可测试的草稿数据。
- [ ] 产品、案例、行业方案、媒体等已有可供首页绑定的测试数据。
- [ ] 页面发布、版本历史、受控预览 Token 的基础接口可用，或已明确需要补充的接口契约。
- [ ] Portal 缓存 Key 与依赖失效规则已登记。

## 3. 实施顺序

### 3.1 建立首页区块模板与默认 Schema

1. 为每个首页区块定义默认 `props`、允许的数据绑定和样式变体。
2. Hero 区块至少支持标题、副标题、背景媒体、主次按钮、按钮跳转类型和替代文本。
3. 指标、AI、能力、产品、方案、案例等列表型区块至少支持标题、摘要、展示数量、排序策略、手动选择和空状态配置。
4. 联系 CTA 至少支持标题、说明、按钮和跳转目标；不允许在页面配置中保存客户线索信息。
5. 提供首页默认区块序列，用于草稿缺失时的安全初始化。

**交付物**：首页区块模板、默认 Schema、组件配置表单定义。

### 3.2 实现编辑画布与属性面板

1. 在 `edit` 模式为每个区块包裹 `EditableSection`。
2. 悬停区块显示编辑、复制、隐藏、删除、上移、下移和拖拽排序控件。
3. 点击编辑后在右侧抽屉按组件类型展示受控表单；表单字段不得允许直接输入脚本、任意 CSS 或未知 JSON。
4. 文本修改即时反映在画布；离开页面前存在未保存修改时提示保存或放弃。
5. 区块新增入口仅在合法 `main` 插槽内展示；新增后插入新实例并打开配置抽屉。
6. 区块复制应深拷贝配置、清理实例 ID、保留合法的数据绑定；复制完成后默认不自动发布。
7. 删除和隐藏必须有明确差异：删除移除草稿 Schema，隐藏保留配置但不在发布页面展示。

**交付物**：首页编辑画布、右侧抽屉、区块选择器、拖拽排序、未保存修改保护。

### 3.3 接入媒体库

1. 提供媒体选择器，支持选择已有媒体、上传新媒体、替换媒体和设置替代文本。
2. 图片字段保存媒体 ID、展示 URL 或由后端派生的可访问 URL、替代文本和受控裁切信息。
3. Hero、Logo、封面等图片必须声明比例或尺寸，以减少 Portal CLS。
4. 上传后仅允许选择已完成处理且满足公开展示条件的媒体。
5. 图片加载失败时展示品牌占位样式；编辑器要提示媒体不可用，而非静默发布破图。

**交付物**：媒体选择器、上传反馈、媒体引用绑定、图片异常提示。

### 3.4 实现内容绑定

1. 为产品、案例、行业方案区块提供手动选择与受控自动选择两种模式。
2. 手动选择模式保存内容 ID 与展示顺序，不复制完整内容实体。
3. 自动选择模式只能提供已定义的过滤条件，例如已发布状态、指定标签、展示数量和排序字段；不开放任意数据库查询表达式。
4. 页面画布加载数据绑定结果时，显示加载、空数据和失败状态。
5. 绑定到未发布、隐藏、删除或不存在内容时，保存或发布前必须报出可理解的校验错误。

**交付物**：内容选择器、绑定 Schema、绑定结果预览、发布前依赖校验。

### 3.5 完成预览与发布链路

1. 点击保存草稿后获取最新草稿版本和哈希。
2. 仅允许已保存草稿生成预览 Token；本地未保存修改必须提示先保存。
3. 预览页面使用同一渲染器和草稿 Schema，显示只读预览标识，禁止编辑操作。
4. 发布前执行：有效锁、页面版本、组件 Schema、必填字段、链接协议、媒体状态、内容依赖、SEO 基础字段校验。
5. 发布成功后生成不可变页面快照和版本记录，发布旧快照保留为历史版本。
6. 发布后失效页面渲染、页面 SEO、首页聚合及受影响列表的 Portal 缓存；采用事务提交后删除和延迟二次删除。
7. 页面回滚从历史版本生成新的已发布版本，不允许直接修改历史快照。

**交付物**：首页预览流程、发布确认、发布校验、缓存失效、版本列表和回滚界面。

### 3.6 处理首页异常状态和可访问性

1. 为编辑器与 Portal 分别处理加载、空数据、媒体失败、保存失败、发布失败、锁失效和网络中断。
2. 编辑工具栏、抽屉、弹窗和确认对话框支持键盘操作、焦点管理和 Escape 关闭。
3. 编辑器不能降低 Portal 无障碍要求：图片替代文本、语义化区块、按钮可访问名称仍必须满足要求。
4. 发布失败时保留草稿和当前编辑上下文，不得清空画布。

## 4. 关键测试清单

- [ ] 管理员可编辑首页 Hero 文本、按钮和背景媒体，并保存为草稿。
- [ ] 管理员可新增、复制、隐藏、删除和拖拽排序首页区块。
- [ ] 仅注册组件可被添加，非法插槽和重复受限组件被阻止。
- [ ] 替换图片后预览、发布页面均显示正确媒体与替代文本。
- [ ] 手动绑定产品、案例和方案后按保存顺序展示。
- [ ] 被下线或删除的关联内容阻止发布，并指出具体依赖项。
- [ ] 保存草稿后普通用户首页内容不变。
- [ ] 预览 Token 无效、过期或撤销后无法读取草稿。
- [ ] 发布后普通用户刷新首页可看到新版本，且旧缓存不会长期返回。
- [ ] 回滚后 Portal 页面、SEO 元数据和相关首页聚合内容一致恢复。
- [ ] 锁失效后管理员无法继续保存或发布，界面提示重新获取编辑权。

## 5. 阶段交付物

- 首页可视化编辑器及桌面端操作说明。
- 首页区块组件库、组件配置 Schema 和默认页面 Schema。
- 媒体选择器与媒体引用校验。
- 内容绑定选择器与依赖校验。
- 首页草稿、预览、发布、版本历史、回滚和缓存失效流程。
- 首页 E2E 测试、视觉回归基线和发布验证记录。

## 6. 阶段退出条件

- [ ] 管理员无需进入分散模块表单，即可完成首页主要内容编辑。
- [ ] 保存草稿、受控预览、发布与回滚均可端到端完成。
- [ ] 普通用户在发布前无法看到草稿，发布后能稳定看到新内容。
- [ ] 首页中的受控区块数量可扩充，且新增区块不破坏页面结构和性能。
- [ ] 页面发布、关联内容和缓存失效均通过接口及 E2E 测试。
- [ ] 本阶段新增接口、错误码和缓存依赖已同步更新项目文档。
