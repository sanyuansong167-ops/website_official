## 3. 实施顺序

### 3.1 全站公共布局与导航

1. 实现 `SiteHeader`：站点 Logo、一级菜单、二级菜单、当前路由高亮、主 CTA 和移动端抽屉。
2. 导航数据只能来自 Portal 站点配置与导航接口；不在前端硬编码菜单、顺序或可见性。
3. 实现并测试四种目标类型：
   - `GROUP`：只展开子菜单，不生成无意义跳转。
   - `INTERNAL_ROUTE`：使用站内路由，并校验以 `/` 开头。
   - `PAGE_ANCHOR`：处理固定 Header 的锚点偏移。
   - `EXTERNAL_LINK`：仅接受 `http/https`，新窗口使用 `noreferrer noopener`。
4. 实现 `SiteFooter`：品牌简介、快捷链接、服务领域、联系方式、社交入口、隐私政策、服务条款和备案号。
5. 公开页面不渲染任何编辑入口；管理员从 `/admin/editor` 进入编辑模式。
6. Header、Footer 的草稿配置必须随对应页面或全局配置发布，不能在普通 Portal 中读取草稿。

**交付物**：全站 Header、Footer、移动菜单、导航目标解析、公共 SEO 默认值和全局编辑配置。

### 3.2 企业实力页面 `/strength`

1. 建立 `StrengthPage` 默认 Schema：页面标题、荣誉区、客户 Logo 墙、实力指标区和联系 CTA。
2. 接入荣誉、客户 Logo、企业实力指标的 Portal 数据源；前端按后端排序直接渲染。
3. 荣誉卡必须有可读文字，客户 Logo 必须提供客户名称替代文本。
4. 指标动画仅在用户未启用减少动态效果时执行；空数据时隐藏无意义区块或显示简洁空状态。
5. 编辑器支持修改区块标题、说明、布局、展示数量、背景主题和 CTA；实体内容编辑留给后续独立内容能力或既有模块。

**交付物**：企业实力公开页、草稿编辑页、预览、发布、回滚和缓存依赖配置。

### 3.3 AI 战略页面 `/ai-strategy`

1. 建立深色主题页面 Schema：战略导语、AI 卡片网格、主 CTA 和辅助内容区。
2. 接入 AI 战略卡片数据；卡片支持图标、英文小标题、中文标题、摘要和详情/联系入口。
3. CTA 跳转到 `/contact` 时，只允许携带合作方向等非敏感上下文；禁止携带姓名、邮箱、电话。
4. 编辑器允许配置页面文案、卡片展示数量、卡片顺序、深浅主题和 CTA；AI 卡片实体内容按照受控数据源展示。
5. 对深色背景校验文字对比度、键盘焦点样式和图标替代文本。

**交付物**：AI 战略公开页、编辑区块、预览和发布链路。

### 3.4 核心能力页面 `/capabilities`

1. 建立能力分类、说明、子项和 CTA 的页面 Schema。
2. 接入能力分类与子项树，按后端数据顺序渲染；子项仅为展示信息，不使用无语义可点击控件。
3. 桌面端采用三栏或响应式网格，移动端单列或适配网格。
4. 空数据时展示简洁状态和联系 CTA，不保留空白大区块。
5. 编辑器支持配置标题、说明、展示方式、子项显示上限、CTA 和区块排序。

**交付物**：核心能力公开页、编辑区块、空状态和发布链路。

### 3.5 产品、案例、行业方案列表页

按顺序实现 `/products`、`/solutions`、`/cases`：

1. 建立三个独立页面 Schema，禁止复用导致不同业务字段混乱的单一“通用列表页”配置。
2. 产品列表展示图标、名称、定位、摘要、状态标签和详情入口；未知状态使用中性样式。
3. 方案列表展示名称、图标、描述、典型客户标签和详情/联系入口；未经后端定义不得把标签实现为筛选查询。
4. 案例列表展示标题、摘要、最多三枚标签、图标和明确详情入口。
5. 三类列表只消费已发布、可见、未删除的 Portal 数据；页面配置只控制展示顺序、数量、标题和样式，不复制完整实体详情。
6. 详情路由在本阶段实现公开展示壳、SEO、404 与返回列表入口；独立详情编辑、富文本、关联内容和实体发布在阶段 04 完成。
7. 编辑器支持列表区块标题、描述、手动选择/受控自动选择、展示数量、布局和 CTA 配置。

**交付物**：三个列表页、三个详情展示壳、相关页面区块编辑能力和缓存依赖清单。

### 3.6 关于我们页面 `/about`

1. 建立页面 Schema，包含创新研发体系、合作高校、重点研发方向、发展历程、价值观和承诺区块。
2. 合作高校 Logo、研发方向、价值观卡片和承诺内容均由 Portal 数据驱动。
3. 时间线按接口返回顺序渲染；桌面端可左右交错，移动端改为单列靠左，不能依赖前端重新猜测排序。
4. 深色研发区、时间线和浅色承诺区分别提供受控主题、间距和标题配置。
5. 编辑器可以新增可重复的图文、时间线、价值观或 CTA 受控区块，但不能脱离注册表创建任意布局代码。

**交付物**：关于我们公开页、区块配置、预览发布回滚和响应式时间线。

### 3.7 联系我们页面 `/contact`

1. 建立联系信息、合作方向、预约表单和确认提示的页面 Schema。
2. 接入公开地址、商务电话、邮箱和合作方向标签；缺失数据时隐藏对应展示项，不显示技术错误。
3. 合作方向点击后可预填表单字段，用户仍可修改；不把线索字段写入 URL、标题、埋点或 `localStorage`。
4. 使用受控表单校验：姓名、公司、邮箱及后端要求的字段；文本按纯文本提交。
5. 提交中禁用重复提交；成功后清空表单并展示确认；失败时保留输入但不在错误提示中回显敏感信息。
6. 正确处理频率限制、网络错误、字段校验和服务错误；客户端校验不替代服务端限流、反垃圾和敏感词策略。
7. 编辑器仅允许修改页面文案、表单前后说明、区块样式和 CTA；不得通过页面 Schema 修改线索存储与安全规则。

**交付物**：联系页面、预约表单、合作方向预填、错误处理和隐私保护测试。

### 3.8 页面构建器动态路由与辅助页面

1. 对所有已发布 `routePath` 提供通用 `PageRenderer` 路由，按发布 Schema 渲染受控区块。
2. 页面 SEO 优先读取已发布页面元数据；未配置时回退站点默认 SEO。
3. 实现产品、案例、方案详情不存在、内容下线或无权限公开时的统一 404 页面。
4. 实现受控预览页：仅凭有效预览 Token 读取草稿，只读展示，设置 `noindex`，过期或撤销后拒绝访问。
5. 实现通用错误页和网络失败状态，保留返回首页或返回列表的可访问入口。
6. 实现隐私政策、服务条款页面：优先使用页面构建器的受控富文本页面；若正式文案未就绪，作为上线阻塞项而非硬编码占位内容。

## 4. 交付物

- 9 个一级导航页面的 Portal 公开页面和 Admin 编辑页面。

- 产品、案例、行业方案列表页与三个公开详情展示壳。
- 404、预览、错误、隐私政策和服务条款辅助页面。

## 5. 全站验收清单

- [ ] 九个一级导航路由均有正常、加载、空数据、请求失败和未找到内容处理。
- [ ] 九个一级导航均只显示发布内容，草稿只在管理员编辑器和有效预览中出现。
- [ ] `/strength`、`/ai-strategy`、`/capabilities`、`/about`、`/contact` 均具有独立页面 Schema 与编辑能力。
- [ ] `/products`、`/solutions`、`/cases` 均具有独立列表配置、已发布数据过滤和详情入口。
- [ ] `/products/{id}`、`/cases/{id}`、`/solutions/{id}` 均存在公开详情展示与统一 404；完整详情编辑在阶段 04 验收。
- [ ] 页脚中的隐私政策、服务条款、备案与外链均按最终配置工作。
- [ ] 联系表单不泄露或持久化个人信息，错误与限流反馈符合安全规范。
- [ ] 所有页面在 `375px`、`768px`、`1024px`、`1440px` 宽度下无横向溢出、遮挡和不可点击控件。
