# 官网可视化编辑模式执行计划

> 文档状态：待实施  
> 版本：v1.0  
> 更新日期：2026-07-14  
> 适用范围：官网 Portal、Admin 后台、页面构建器、产品/案例/行业方案内容管理。  
> 关联文档：`docs/官网前端实现说明.md`、`docs/接口文档.md`、`docs/重构接口文档.md`、`docs/数据库设计说明.md`、`docs/错误码说明.md`。

## 1. 建设目标

将官网内容管理从分散的传统表单后台升级为**可视化编辑模式**：管理员在与普通用户官网展示效果一致的页面中编辑内容、调整区块、预览草稿并发布；普通用户始终只能看到已发布内容。

本期目标：

1. 管理员通过受保护的编辑器管理官网页面，不在公开页面暴露编辑入口。
2. 强制执行“草稿 → 预览 → 发布 → 回滚”流程，不允许保存后直接影响普通用户。
3. 管理员可在受控范围内新增、复制、删除、隐藏、排序和配置页面区块。
4. 产品、案例、行业方案提供独立详情编辑器，分别维护详情内容、媒体、SEO、关联关系和发布状态。
5. 同一页面或同一内容实体同一时间仅允许一名管理员编辑。
6. 发布后自动失效 Portal 缓存，普通用户重新访问即可看到最新版本。

## 2. 已确认的实施边界

| 项目 | 本期决策 |
| --- | --- |
| 公开页面 | 普通用户仅访问 Portal 已发布页面，不显示编辑按钮、草稿、版本或管理信息。 |
| 管理员入口 | 管理员从 Admin 后台进入受保护的可视化编辑器。 |
| 发布流程 | 强制草稿、预览、发布；仅发布成功后对普通用户生效。 |
| 并发编辑 | 禁止同一资源多人同时编辑，使用服务端独占编辑锁。 |
| 页面扩展 | 可增加区块数量、自由组合受控组件；不支持任意 HTML、JavaScript、CSS 或未注册组件。 |
| 独立内容 | 产品、案例、行业方案均有独立详情编辑器和版本管理。 |
| 编辑终端 | 首期仅支持桌面端管理员编辑；移动端编辑器后续单独规划。 |
| 数据边界 | Portal 仅调用 `/portal/api/**`；编辑器仅调用受权限保护的 `/admin/api/**`。 |

## 3. 总体架构

### 3.1 页面访问与数据流

```text
普通用户
  -> Portal 页面（/、/products、/cases 等）
  -> /portal/api/**
  -> 仅读取已发布快照与已发布内容

管理员
  -> Admin 后台
  -> /admin/editor?routePath=/目标页面
  -> /admin/api/**
  -> 读取草稿、持有编辑锁、保存草稿、预览、发布、回滚
```

推荐路由：

```text
/admin/editor?routePath=/
/admin/editor?routePath=/about
/admin/editor?routePath=/products
/admin/editor?routePath=/cases

/admin/content/products/{id}
/admin/content/cases/{id}
/admin/content/solutions/{id}
```

公开页面继续使用：

```text
/
/products
/products/{id}
/cases
/cases/{id}
/solutions
/solutions/{id}
```

### 3.2 同组件、不同模式渲染

Portal 页面与编辑器必须复用同一套展示组件、设计 Token 和响应式规则，避免“编辑预览效果”与“实际官网效果”不一致。

```text
view：普通用户浏览已发布内容
preview：管理员草稿预览，只读
edit：管理员可视化编辑
```

`edit` 模式在官网组件外增加选中状态、区块工具栏、拖拽手柄和右侧属性面板；`view` 模式不得加载或渲染这些管理能力。

### 3.3 前端模块边界

```text
components/
├── portal/            # 公开官网展示组件，仅依赖 Portal ViewModel
├── editor/            # 编辑工具栏、区块包装器、属性抽屉、拖拽能力
└── shared/            # Button、Modal、MediaPicker、Toast、EmptyState 等

features/
├── portal-page/       # 已发布页面查询与渲染
├── page-editor/       # 页面草稿、编辑锁、保存、预览、发布、版本
├── content-editor/    # 产品、案例、行业方案独立详情编辑
└── media-library/     # 媒体选择、上传、替换、引用管理

services/
├── portal-api.ts      # 仅 /portal/api/**
├── admin-api.ts       # 仅 /admin/api/**
└── http.ts            # 认证、CSRF、错误归一化、traceId
```

## 4. 权限与安全要求

### 4.1 普通用户

- 只能访问已发布 Portal 页面和公开详情。
- 不显示“进入编辑”、区块边框、草稿状态、编辑锁、版本历史或管理员信息。
- 无法通过 URL、缓存键、接口参数或前端代码读取草稿、预览 Token 和后台数据。

### 4.2 管理员

- `/admin/editor` 与 `/admin/content/**` 必须校验已登录的 `ADMINISTRATOR` 权限。
- 所有新增、保存、删除、发布、回滚、强制解锁接口必须进行后端权限校验与 CSRF 校验。
- 前端隐藏按钮不能作为权限边界；即使调用者手工构造请求，后端也必须拒绝越权请求。
- 编辑、发布、回滚、强制解锁、查看线索敏感信息等行为必须写审计日志。

### 4.3 内容安全

- 区块组件必须来自受控组件注册表。
- 富文本必须在保存和输出时进行白名单净化。
- 外部链接仅允许 `http`、`https` 协议；新标签打开必须使用安全属性。
- 媒体配置优先保存媒体 ID，不以用户输入的任意 URL 作为唯一来源。
- Portal 响应不得返回草稿、编辑锁、审计快照、内部存储路径、Token 或敏感线索数据。

## 5. 页面与区块管理

### 5.1 页面模型

每个页面至少包含以下逻辑字段：

```text
pageKey
pageName
routePath
seo
layout
draftSchema
publishedSnapshot
status
version
updatedAt
```

页面草稿与发布快照必须分离。Portal 只读取发布快照；Admin 编辑器读取草稿和编辑状态。

### 5.2 受控组件注册表

首期组件类型建议包括：

```text
HeroSection
MetricsSection
RichTextSection
ImageTextSection
ProductGridSection
CaseGridSection
SolutionGridSection
AiCardSection
CapabilitySection
TimelineSection
LogoWallSection
ContactCtaSection
ContactFormSection
FaqSection
DividerSection
```

管理员可选择、组合和重复使用上述受控组件；后续可增加组件类型，但不得将未知组件直接发布到 Portal。

### 5.3 受控插槽与区块数量

页面使用受控插槽组织内容：

```text
header
main
footer
```

本期规则：

- `header`、`footer` 为全局受控组件，首期不允许任意删除。
- `main` 插槽允许添加多个区块，数量可扩充。
- 支持新增、复制、上移、下移、拖拽排序、隐藏和删除区块。
- 部分页面可保留最低结构要求，例如联系页至少保留一个联系信息区块或预约表单区块。
- 发布前必须校验组件是否允许放入当前插槽、必填配置是否完整、是否存在非法链接和无效媒体。

### 5.4 区块数据模型

```json
{
  "id": "section-unique-id",
  "component": "ProductGridSection",
  "visible": true,
  "sortOrder": 30,
  "props": {
    "title": "产品矩阵",
    "subtitle": "面向多行业的数据智能能力"
  },
  "dataBinding": {
    "source": "PRODUCT",
    "displayMode": "MANUAL",
    "selectedIds": [1, 3, 6]
  },
  "style": {
    "theme": "light",
    "layout": "three-columns",
    "spacing": "large"
  }
}
```

约束：

- `component` 必须存在于注册表。
- `props`、`style`、`dataBinding` 必须满足当前组件 Schema。
- `style` 只能使用预定义主题、布局和间距 Token，不允许任意 CSS。
- `visible=false` 的区块不向 Portal 输出。
- 遇到未知组件或非法配置时，编辑器显示错误；Portal 安全跳过该区块，避免整页崩溃。

## 6. 管理员编辑体验

### 6.1 编辑器布局

```text
顶部编辑工具栏
├── 当前页面与页面切换
├── 设备宽度预览
├── 保存草稿
├── 预览
├── 发布
├── 历史版本
└── 退出编辑

中间画布
└── 与官网一致的页面展示效果

右侧属性面板
├── 区块基础配置
├── 文本与按钮
├── 媒体选择与替代文本
├── 数据来源与内容选择
├── 受控样式变体
└── 可见性与排序
```

### 6.2 区块操作

管理员悬停区块时显示：

```text
编辑 | 复制 | 上移 | 下移 | 拖拽排序 | 显示/隐藏 | 删除
```

- 修改内容后在画布即时预览，但未保存前不写入服务端。
- “保存草稿”后才持久化草稿。
- 删除操作必须二次确认。
- 复制操作生成新的区块实例 ID，禁止两个区块共享同一实例 ID。
- 新增区块从受控组件选择器进入，并自动插入当前选中的合法位置。

### 6.3 媒体编辑

所有图片、图标、封面和附件统一使用媒体库：

```text
选择已有媒体 | 上传新媒体 | 替换媒体 | 设置替代文本 | 设置受控裁切比例
```

必须满足：

- 保存媒体 ID 与必要的展示属性。
- 业务图片提供有意义的替代文本；装饰图片可标记为装饰性。
- 富文本媒体引用通过引用清单维护，不仅依赖 HTML 内容扫描。
- 失效、未发布或无权限媒体不得在 Portal 输出。

## 7. 产品、案例与行业方案独立详情编辑器

### 7.1 产品详情编辑器

地址：`/admin/content/products/{id}`。

编辑范围：

- 产品名称、短标题、摘要、发布状态和排序。
- Logo、封面、图集、详情头图。
- 富文本正文、能力亮点、适用行业。
- 关联案例、关联行业方案、推荐产品。
- SEO 标题、关键词和描述。
- 草稿、预览、发布、版本历史与回滚。

公开详情页：`/products/{id}`，仅返回已发布产品。

### 7.2 案例详情编辑器

地址：`/admin/content/cases/{id}`。

编辑范围：

- 案例标题、客户名称、行业、摘要和状态。
- 项目背景、解决方案、成果、富文本正文。
- 封面、图片集、关联产品、推荐案例。
- SEO 配置、草稿、预览、发布、版本历史与回滚。

公开详情页：`/cases/{id}`，仅返回已发布案例。

### 7.3 行业方案详情编辑器

地址：`/admin/content/solutions/{id}`。

编辑范围：

- 方案名称、摘要、行业标签和状态。
- 行业痛点、解决思路、核心能力、应用场景和典型客户。
- 图标、封面、正文、关联产品和案例。
- SEO 配置、草稿、预览、发布、版本历史与回滚。

公开详情页：`/solutions/{id}`，仅返回已发布行业方案。

### 7.4 页面区块与内容实体的职责

- 页面区块负责展示方式：标题、布局、样式、展示数量、手工选择内容或自动筛选条件。
- 内容实体负责业务详情：正文、媒体、SEO、关联关系、发布状态与版本。
- 区块不得复制保存完整产品、案例或方案详情，避免内容来源不一致。

## 8. 独占编辑锁与并发控制

### 8.1 锁定对象

以下对象必须独占编辑：

```text
页面定义
产品详情
案例详情
行业方案详情
站点基础配置
导航配置
```

编辑不同资源的管理员可以并行工作；编辑同一资源时必须互斥。

### 8.2 编辑锁模型

```text
resourceType
resourceId
lockedBy
lockedByName
lockedAt
lastHeartbeatAt
expiresAt
```

流程：

1. 管理员进入编辑器时请求创建编辑锁。
2. 锁创建成功才允许进入编辑状态。
3. 已锁定时其他管理员仅可查看，不可修改，并显示锁定账号与占用提示。
4. 编辑器周期性发送心跳续期。
5. 保存、发布、退出编辑时主动释放锁。
6. 浏览器异常关闭或会话中断时，锁超时自动释放。
7. 仅拥有更高管理权限的账号可强制解锁；强制解锁必须填写原因并写审计日志。

### 8.3 发布前二次校验

发布时后端必须再次验证：

- 当前管理员仍持有有效编辑锁。
- 当前草稿版本未被异常更新。
- 页面或内容实体符合发布校验。
- 关联内容、链接、媒体和 SEO 配置有效。

## 9. 草稿、预览、发布与回滚

### 9.1 草稿

- 管理员保存草稿后，服务端保存草稿 Schema 或内容草稿及版本号。
- 草稿仅管理员可见，不刷新 Portal 缓存。
- 草稿保存记录编辑人、更新时间、变更说明和版本。
- 发生版本或锁冲突时必须返回稳定错误码并提示刷新，不允许静默覆盖。

### 9.2 预览

- 管理员为已保存草稿生成短期受控预览 Token。
- 预览 Token 绑定资源、草稿哈希、操作人和过期时间，可撤销。
- 预览页面只读，并使用与 Portal 一致的组件渲染。
- 预览页面设置 `noindex`，不得被搜索引擎收录。
- 日志中不得输出完整 Token。

### 9.3 发布

发布流程：

1. 前端提示“发布将立即更新官网公开内容”。
2. 后端执行权限、CSRF、编辑锁、版本和 Schema 校验。
3. 生成不可变发布快照与新版本记录。
4. 更新资源的已发布状态和发布时间。
5. 写入审计日志。
6. 事务提交后失效关联 Portal 缓存，并执行延迟二次删除。
7. Portal 后续读取最新发布快照。

### 9.4 回滚

- 管理员可查看历史发布版本的版本号、发布人、发布时间和变更摘要。
- 回滚基于历史版本生成新的发布版本，不删除或覆盖历史记录。
- 回滚后同样失效关联 Portal 缓存并记录审计日志。

## 10. 接口与数据实施要求

### 10.1 已有页面构建器能力

现有页面构建器接口是本计划的基础，至少包括：

```text
组件模板列表与详情
页面定义创建、查询、更新、删除
页面草稿读取与保存
受控预览 Token 生成、访问、撤销
页面发布、回滚、版本历史
Portal 已发布页面渲染与 SEO 元数据读取
```

接口实现和错误码必须持续同步到 `docs/接口文档.md` 与 `docs/重构接口文档.md`。

### 10.2 本期需新增或明确的能力

| 能力 | 建议接口 | 说明 |
| --- | --- | --- |
| 页面加锁 | `POST /admin/api/page-builder/pages/{pageId}/lock` | 创建独占编辑锁。 |
| 页面心跳 | `POST /admin/api/page-builder/pages/{pageId}/lock/heartbeat` | 续期当前管理员的编辑锁。 |
| 页面解锁 | `DELETE /admin/api/page-builder/pages/{pageId}/lock` | 主动释放编辑锁。 |
| 强制解锁 | `POST /admin/api/page-builder/pages/{pageId}/lock/force-release` | 仅高权限管理员可用，必须记录原因。 |
| 内容加锁 | `/admin/api/products/{id}/lock` 等 | 产品、案例、方案采用同等互斥机制。 |
| 内容草稿与发布 | 对应资源的草稿、预览、发布、版本、回滚接口 | 独立详情编辑器必须支持完整发布链路。 |
| 行业方案详情 | `GET /portal/api/industry-solutions/{id}` | 若尚未实现，需补充公开详情接口。 |

所有 Admin 写接口必须包含版本校验，并返回统一响应结构、稳定错误码与 `traceId`。

### 10.3 缓存失效

发布、回滚或影响已发布内容的写操作提交后，必须按依赖关系失效：

```text
页面渲染缓存
页面 SEO 缓存
首页聚合缓存
产品列表与详情缓存
案例列表与详情缓存
行业方案列表与详情缓存
导航与站点配置缓存
```

遵循已有缓存机制：事务提交后删除缓存、延迟二次删除、缓存失败仅记录日志并由 TTL 兜底；缓存失效失败与重建失败需可追踪。

## 12. 验收标准

### 12.1 功能验收

- [ ] 普通用户只看到已发布 Portal 内容，不存在编辑入口或编辑器资源暴露。
- [ ] 管理员可通过后台进入受保护的可视化编辑器。
- [ ] 管理员可以在页面画布中编辑受控区块并实时查看本地效果。
- [ ] 管理员可新增、复制、排序、隐藏和删除受控区块，且区块数量可扩充。
- [ ] 页面区块只能使用注册组件和合法插槽，不能输入或发布任意脚本和 CSS。
- [ ] 页面、产品、案例、行业方案均支持草稿、预览、发布、版本历史和回滚。
- [ ] 产品、案例、行业方案均具备独立详情编辑器。
- [ ] 发布后，相关列表、详情、页面内容和 SEO 元数据正确更新。

### 12.2 并发与数据一致性验收

- [ ] 同一页面同一时刻仅允许一名管理员进入编辑状态。
- [ ] 同一产品、案例或方案同一时刻仅允许一名管理员编辑。
- [ ] 锁占用、锁超时、主动解锁和强制解锁均有明确提示与审计记录。
- [ ] 管理员不持有有效锁时无法保存、发布或回滚。
- [ ] 保存、发布和回滚不会静默覆盖其他有效版本。

### 12.3 安全验收

- [ ] Admin 路由和全部写接口均校验 `ADMINISTRATOR` 权限与 CSRF。
- [ ] Portal 接口不返回草稿、预览 Token、编辑锁、审计数据或内部字段。
- [ ] 预览 Token 具备有效期、可撤销性与最小权限，日志不输出完整 Token。
- [ ] 富文本、外部链接、媒体和区块配置均经白名单或 Schema 校验。
- [ ] 错误响应不泄露 SQL、堆栈、文件路径、密钥、Token 或敏感个人信息。

### 12.4 性能与体验验收

- [ ] Portal 首屏不加载管理编辑器代码、草稿数据或 Admin API 请求。
- [ ] 编辑器首期在桌面端稳定可用；移动端不作为本期编辑交付范围。
- [ ] 发布后缓存按照事务提交后删除和延迟二次删除规则生效。
- [ ] 页面加载、骨架屏、空状态、失败重试、404 和网络异常处理符合前端实施说明。
- [ ] 官网在 `375px`、`768px`、`1024px`、`1440px` 宽度下保持正常展示；管理员实际编辑操作首期仅承诺桌面端。

### 12.5 文档与测试验收

- [ ] 新增或修改的接口同步更新 `docs/接口文档.md`、`docs/重构接口文档.md`。
- [ ] 新增或修改的错误码同步更新 `docs/错误码说明.md`。
- [ ] 涉及表结构、锁、草稿、版本、审计和媒体引用的变更同步更新 `docs/数据库设计说明.md` 与迁移脚本。
- [ ] 接口测试断言业务错误码，不仅断言 HTTP 状态码。
- [ ] 覆盖编辑锁、草稿隔离、预览 Token、发布、缓存失效、回滚和越权访问的自动化测试。

## 13. 上线准入条件

满足以下全部条件后，才可向生产环境开放管理员可视化编辑能力：

1. 页面编辑器、三类独立详情编辑器和发布链路完成 P0 验收。
2. 普通用户无法读取任何草稿或后台数据的安全测试通过。
3. 独占编辑锁及异常超时释放测试通过。
4. 发布、回滚和缓存失效在预发布环境完成端到端验证。
5. 关键操作审计日志可查询，错误日志带 `traceId`。
6. 数据库迁移脚本可在空库执行，并具有风险变更说明。
7. 接口、错误码、数据库和前端文档均已同步更新并完成 Review。
8. 已定义生产环境发布窗口、回滚负责人和异常处置流程。

## 14. 后续扩展（不属于本期交付）

- 移动端管理员编辑器。
- 内容审批流、多人协作和评论机制。
- 定时发布、定时下线和内容日历。
- 页面模板市场、区块模板复制和跨页面复用。
- 细粒度角色权限，如内容编辑、审核人、发布人分离。
- 更高级的版本差异对比、变更合并和编辑冲突解决机制。
