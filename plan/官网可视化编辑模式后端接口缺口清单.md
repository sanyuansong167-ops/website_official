# 官网可视化编辑模式：确定版后端接口契约清单

> 版本：v2.0
> 更新日期：2026-07-14
> 适用范围：官网 Portal、Admin 后台、页面构建器、产品、案例、行业方案。
> 关联文档：`docs/接口文档.md`、`docs/重构接口文档.md`、`docs/数据库设计说明.md`、`docs/错误码说明.md`、`plan/官网可视化编辑模式执行计划.md`。

## 1. 文档定位与契约优先级

本文件定义官网可视化编辑模式尚未实现及需要收紧的后端接口契约。接口实现完成后，必须同步写入 `docs/接口文档.md`；这份接口文档是已实现接口的唯一事实来源。

前端请求地址、方法、路径参数、查询参数、Header、请求体、响应体和错误处理必须严格使用本文件定义的契约。前端不得调用传统 CRUD 接口完成草稿保存、预览、发布、回滚、下线、删除和编辑锁操作。

契约优先级如下：

1. 已实现接口使用 `docs/接口文档.md` 与 `docs/重构接口文档.md` 的真实路径。
2. 本文件定义缺失接口及已实现接口的新增安全约束。
3. 后端代码实现完成后，以同步更新后的两份正式接口文档为唯一事实来源。

## 2. 固定请求约定

### 2.1 鉴权与 Header

所有 `/admin/api/**` 接口使用现有 Cookie Session 鉴权，并使用以下 Header：

| Header | 使用范围 | 规则 |
| --- | --- | --- |
| `X-CSRF-Token` | 所有 Admin 写请求 | 必填，值由 `GET /admin/api/auth/csrf` 获取。 |
| `X-Editor-Lock-Token` | 草稿保存、预览创建、预览撤销、发布、回滚、下线、删除 | 必填，值由对应资源的加锁接口返回。 |
| `Content-Type: application/json` | JSON 请求体 | 必填。 |

`lockToken` 不出现在 URL、查询参数、页面 HTML、浏览器持久化存储、埋点、日志和 Referer 中。前端仅在受保护编辑器内存状态中保留该值。

### 2.2 通用响应结构

所有接口返回现有统一响应结构：

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {},
  "traceId": "string"
}
```

所有错误响应保留 `code`、`message`、`data` 与 `traceId`。字段级校验错误放入 `data.fieldErrors`。新增错误码在实现时登记到 `docs/错误码说明.md`，并在正式接口文档中登记数值、名称和 HTTP 状态。

### 2.3 资源与版本规则

| 资源 | 资源键 | 版本字段 | 锁 Header |
| --- | --- | --- | --- |
| 页面构建器页面 | `PAGE:{pageId}` | `version` | `X-Editor-Lock-Token` |
| 产品详情 | `PRODUCT:{id}` | `version` | `X-Editor-Lock-Token` |
| 案例详情 | `CASE:{id}` | `version` | `X-Editor-Lock-Token` |
| 行业方案详情 | `INDUSTRY_SOLUTION:{id}` | `version` | `X-Editor-Lock-Token` |

`version` 放在草稿、发布、回滚、下线和删除的 JSON 请求体中。创建资源草稿、创建锁、锁心跳、主动释放锁和强制释放锁不携带 `version`。强制释放锁不携带 `X-Editor-Lock-Token`，服务端只校验管理员身份、CSRF、强制解锁权限和解锁原因。其余受保护写请求同时校验管理员身份、CSRF、资源锁和 `version`。服务端锁校验结果为唯一有效结果。

## 4. 编辑锁接口

编辑锁使用资源嵌套路径。集中式锁路由不属于本期接口契约。页面、产品、案例、行业方案使用相同字段、HTTP 状态和行为语义。

### 4.1 统一锁响应

加锁、查询锁和心跳接口的 `data` 使用以下字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `resourceType` | String | `PAGE`、`PRODUCT`、`CASE`、`INDUSTRY_SOLUTION`。 |
| `resourceId` | Long | 被锁定资源 ID。 |
| `editable` | Boolean | 当前管理员是否持有有效锁。 |
| `lockToken` | String | 仅在当前管理员成功创建锁时返回。查询锁状态与冲突响应不返回该字段。 |
| `ownerDisplayName` | String | 脱敏管理员显示名。 |
| `acquiredAt` | ISO-8601 String | 获锁时间。 |
| `expiresAt` | ISO-8601 String | 锁到期时间。 |
| `heartbeatIntervalSeconds` | Integer | 前端心跳间隔秒数。 |
| `forceUnlockAllowed` | Boolean | 当前管理员是否具备强制解锁权限。 |

### 4.2 页面锁接口

| 前端能力 | 方法 | 固定路径 | 请求体 | 成功响应 |
| --- | --- | --- | --- | --- |
| 查询页面锁状态 | `GET` | `/admin/api/page-builder/pages/{pageId}/lock` | 无 | 统一锁响应，不含 `lockToken`。 |
| 创建页面锁 | `POST` | `/admin/api/page-builder/pages/{pageId}/lock` | `{ "editorSessionRemark": "string" }` | 统一锁响应，包含 `lockToken`。 |
| 页面锁心跳 | `POST` | `/admin/api/page-builder/pages/{pageId}/lock/heartbeat` | 无 | 统一锁响应，包含新的 `expiresAt`。 |
| 主动释放页面锁 | `DELETE` | `/admin/api/page-builder/pages/{pageId}/lock` | 无 | `data: null`。 |
| 强制释放页面锁 | `POST` | `/admin/api/page-builder/pages/{pageId}/lock/force-release` | `{ "reason": "string" }` | 使用 `X-CSRF-Token`，不使用 `X-Editor-Lock-Token`；响应为 `{ "auditId": "long", "releasedAt": "ISO-8601 string" }`。 |

页面锁接口逐项对应 `plan/官网可视化编辑模式/01-编辑器基础设施与独占锁.md` 和 `plan/官网可视化编辑模式执行计划.md` 的前端请求地址。

产品、案例、行业方案锁接口的 `POST`、`GET`、心跳、释放和强制释放请求与页面锁接口使用相同请求体、Header 和响应字段。

### 4.6 锁状态与失败处理

| 场景 | HTTP 状态 | `data` 字段 | 前端动作 |
| --- | --- | --- | --- |
| 已被其他管理员锁定 | `409` | `ownerDisplayName`、`expiresAt`、`forceUnlockAllowed` | 进入只读状态。 |
| 锁已过期 | `409` | `resourceType`、`resourceId` | 禁用受保护操作并重新获取锁。 |
| 锁归属不匹配 | `403` | 空对象 | 禁用受保护操作。 |
| 无强制解锁权限 | `403` | 空对象 | 隐藏强制解锁操作。 |

锁失效后，前端立即禁用保存、预览、发布、回滚、下线、删除、区块新增、区块删除、区块排序和详情字段修改。前端保留未保存的内存输入，重新获取锁后重新读取服务端草稿并显示差异。

## 5. 页面构建器草稿、预览、发布与回滚

### 5.1 保存页面草稿

`PUT /admin/api/page-builder/drafts/{pageId}` 使用 `X-CSRF-Token` 与 `X-Editor-Lock-Token`。

```json
{
  "schemaJson": {},
  "editorSessionRemark": "更新首页首屏标题",
  "version": 12
}
```

服务端校验页面锁、草稿版本、受控组件白名单、合法插槽、区块数量规则、媒体引用、链接协议、富文本净化和页面依赖。成功响应返回 `PageDraftVO`，包含 `id`、`pageId`、`schemaJson`、`schemaHash`、`editorSessionRemark`、`version`、`updatedAt`。

### 5.2 创建与访问页面预览

`POST /admin/api/page-builder/drafts/{pageId}/previews` 使用 `X-CSRF-Token` 与 `X-Editor-Lock-Token`。

```json
{
  "schemaHash": "string"
}
```

成功响应固定返回：

```json
{
  "previewToken": "string",
  "previewUrl": "/admin/preview/pages/{pageId}/{previewToken}",
  "expiresAt": "ISO-8601 string"
}
```

预览数据读取地址固定为 `GET /portal/api/page-builder/previews/{previewToken}`。该接口必须校验当前请求携带的有效管理员 Session、Token 绑定的创建管理员、资源 ID、资源类型和草稿哈希。接口返回 `Cache-Control: no-store` 与 `Referrer-Policy: no-referrer`。未登录用户、普通 Portal 用户、Token 过期用户、Token 撤销用户和非创建管理员均返回 `401`。

预览页面固定使用 `/admin/preview/pages/{pageId}/{previewToken}` 路由。该路由仅在 Admin 前端加载，Portal 导航、Portal HTML 和公开页面不输出预览入口。

`DELETE /admin/api/page-builder/previews/{previewToken}` 使用 `X-CSRF-Token` 与 `X-Editor-Lock-Token`，成功响应为 `data: null`。撤销操作只允许预览创建管理员执行。预览 Token 在到期后由服务端自动失效。

### 5.3 发布页面

`POST /admin/api/page-builder/pages/{pageId}/publish` 使用 `X-CSRF-Token` 与 `X-Editor-Lock-Token`。

```json
{
  "changeSummary": "发布首页首屏标题更新",
  "version": 12
}
```

服务端校验页面锁、草稿版本、Schema、引用实体状态、媒体引用、发布快照和缓存失效计划。成功响应返回 `PageVersionVO`。事务提交后失效页面渲染缓存、页面 SEO 缓存、引用页面缓存和关联列表缓存。

### 5.4 回滚页面

`POST /admin/api/page-builder/pages/{pageId}/rollback` 使用 `X-CSRF-Token` 与 `X-Editor-Lock-Token`。

```json
{
  "versionId": 1001,
  "version": 12,
  "changeSummary": "回滚至版本 1001"
}
```

回滚生成新的发布版本与新的草稿版本，不覆盖历史版本。成功响应返回新 `PageVersionVO`。页面版本列表继续使用 `GET /admin/api/page-builder/pages/{pageId}/versions`。

## 7. 三类独立详情编辑器接口

```json
{
  "id": 1,
  "draft": {},
  "draftHash": "string",

}
```

创建详情草稿使用 `X-CSRF-Token`。获取草稿、查询版本、访问预览和查询锁状态不使用 CSRF。锁创建、锁心跳、主动释放锁使用本文件第 4 节的锁请求规则。保存草稿、创建预览、撤销预览、发布、回滚和下线使用 `X-CSRF-Token` 与 `X-Editor-Lock-Token`。强制释放锁使用 `X-CSRF-Token`，不使用 `X-Editor-Lock-Token`。访问详情预览还必须校验有效管理员 Session、预览 Token 的创建管理员、资源类型、资源 ID 和草稿哈希。详情预览响应返回 `Cache-Control: no-store` 与 `Referrer-Policy: no-referrer`。

`POST /admin/api/products`、`PUT /admin/api/products/{id}`、`DELETE /admin/api/products/{id}`、案例对应传统 CRUD 与行业方案对应传统 CRUD 不属于详情编辑器调用范围。详情编辑器只使用本节接口。

## 9. 引用关系、发布联动与缓存

发布、回滚、下线和删除操作在数据库事务内检查引用关系。服务端维护页面区块、产品、案例、行业方案和媒体之间的引用记录。

| 操作 | 服务端固定行为 |
| --- | --- |
| 发布页面 | 校验被引用详情已经发布且可见；提交后失效页面渲染、页面 SEO 与关联列表缓存。 |
| 发布详情 | 校验关联实体和媒体可用；提交后失效当前详情、所属列表、引用页面渲染、引用页面 SEO 与关联详情缓存。 |
| 下线详情 | 已发布页面存在强引用时返回 `409`，不改变详情状态。 |
| 删除详情 | 已发布页面存在强引用时返回 `409`，不执行删除。 |
| 回滚详情 | 回滚结果破坏已发布页面引用时返回 `409`，不生成新发布版本。 |
| 删除媒体 | 存在已发布快照引用时返回 `409`，不执行删除。 |

缓存失效使用现有事务提交后删除与延迟二次删除机制。缓存删除失败记录审计事件、监控事件和可重试任务；发布结果保持成功状态，Portal 缓存 TTL 负责最终收敛。
